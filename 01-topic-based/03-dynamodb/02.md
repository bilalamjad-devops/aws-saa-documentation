2. Question

Category: CSAA – Design High-Performing Architectures

A popular social network is hosted in AWS and is using a Amazon DynamoDB table as its database. There is a requirement to implement a ‘follow’ feature where users can subscribe to certain updates made by a particular user and be notified via email.

Which of the following is the most suitable solution to implement to meet the requirement?

- `Enable DynamoDB Stream and create an AWS Lambda trigger, as well as the IAM role which contains all of the permissions that the Lambda function will need at runtime. The data from the stream record will be processed by the Lambda function which will then publish a message to Amazon SNS Topic that will notify the subscribers via email.`
- Create an AWS Lambda function that uses DynamoDB Streams Amazon Kinesis Adapter which will fetch data from the DynamoDB Streams endpoint. Set up an Amazon SNS Topic that will notify the subscribers via email when there is an update made by a particular user.
- Using the Amazon Kinesis Client Library (KCL), write an application that leverages on DynamoDB Streams Kinesis Adapter that will fetch data from the DynamoDB Streams endpoint. When there are updates made by a particular user, notify the subscribers via email using Amazon SNS.
- Set up a DAX cluster to access the source DynamoDB table. Create a new DynamoDB trigger and an AWS Lambda function. For every update made in the user data, the trigger will send data to the Lambda function which will then notify the subscribers via email using Amazon SNS.

Excellent. This is **one of the most important DynamoDB questions** for the SAA exam because it introduces **DynamoDB Streams**.

You'll see this concept many times. Let's understand it from scratch.

---

# First, forget AWS.

Imagine you own Instagram.

You have a table:

```text
Users
```

| User  | Followers |
| ----- | --------- |
| Bilal | 120       |
| Ali   | 450       |

Now Bilal posts:

> "I passed AWS SAA!"

What should happen?

1. Save the post.
2. Notify Bilal's followers.

Question:

**How does AWS know that Bilal posted something?**

---

# Answer:

It needs something that says:

> "Hey! Someone changed the database."

That is exactly what **DynamoDB Streams** does.

---

# What is DynamoDB Streams?

Think of it as a **change log**.

Whenever something changes in DynamoDB:

* New item inserted
* Existing item updated
* Item deleted

DynamoDB Streams records that event.

Example:

Your table:

```text
User: Bilal

Followers: 100
```

Later someone follows you.

Now:

```text
Followers: 101
```

DynamoDB Streams records:

```text
Bilal

Followers

100 → 101
```

It doesn't change the table.

It simply says:

> "A change happened."

---

# Think of it like CCTV 🎥

Your database is a shop.

Every time someone enters:

CCTV records it.

Every time someone leaves:

CCTV records it.

The CCTV doesn't stop or change customers.

It simply records events.

DynamoDB Streams is that CCTV.

# Why Lambda?

Because Streams only says:

> "Something changed."

It does **not** send emails.

It needs someone to do work.

That someone is:

Lambda.

Think:

Streams = Alarm

Lambda = Person responding to the alarm

---

# Why SNS?

SNS is made for notifications.

Email

SMS

Mobile Push

HTTP

Perfect.

# Why not DAX?

This is a common exam trap.

What is DAX?

DynamoDB Accelerator.

Notice the name:

Accelerator.

Its only job:

Make reads faster.

It is basically a cache.

It has nothing to do with:

* Events
* Notifications
* Emails
* Streams

So if you see:

> Notify

or

> React to database changes

Never think DAX.

Think:

**Streams.**

---

# Why not Kinesis Adapter?

The question mentions:

> DynamoDB Streams Kinesis Adapter

This confuses almost everyone.

For the SAA exam, remember this:

There are **two ways** to consume DynamoDB Streams.

### Way 1 (Simple - Most Common)

```text
Streams

↓

Lambda
```

AWS handles everything.

This is what the SAA exam almost always expects.

---

### Way 2 (Advanced)

```text
Streams

↓

Kinesis Adapter

↓

Your own application
```

You write and manage your own consumer.

This is more operationally complex.

---

The exam almost always prefers the **managed AWS solution**:

> Streams → Lambda → SNS

Wrong.

# DynamoDB Features Learned So Far

| Feature        | Purpose                                               |
| -------------- | ----------------------------------------------------- |
| Auto Scaling   | Automatically adjusts provisioned read/write capacity |
| Streams        | Captures inserts, updates, and deletes                |
| Lambda Trigger | Automatically runs code when a stream record appears  |
| SNS            | Sends notifications (email, SMS, etc.)                |
| DAX            | Speeds up reads with in-memory caching                |

The question says:

This is actually a distraction.

The exam doesn't care about the social media app. It cares about this pattern:

That pattern is almost always:

**DynamoDB Streams → Lambda**.

7. Question

Category: CSAA – Design Resilient Architectures

A company is running a web application on AWS. # DynamoDB Across Regions

Question:

How do we replicate DynamoDB to another Region?

There are two similar names.

### 1. Global Secondary Index (GSI)

Purpose?

Faster queries.

NOT disaster recovery.

---

### 2. Global Table ✅

Purpose?

Replicate DynamoDB across Regions.

Example:

```text
us-east-1

↓

Global Table

↓

us-west-2
```

Both regions have the data.

If one Region dies,

the other already has the data.

### One small correction to your mental model

Don't confuse **Global Table** and **Global Secondary Index (GSI)**:

* **Global Table** = **Replication** across Regions for high availability and disaster recovery.
* **Global Secondary Index (GSI)** = An **additional index** on the same table that lets you query the data using a different partition key/sort key. It has **nothing to do with multiple Regions**.

This distinction appears frequently in the SAA exam.

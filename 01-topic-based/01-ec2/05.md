5. Question

Category: CSAA – Design Secure Architectures

A payment processing company plans to migrate its on-premises application to an Amazon EC2 instance. An IPv6 CIDR block is attached to the company’s Amazon VPC. Strict security policy mandates that the production VPC must only allow outbound communication over IPv6 between the instance and the internet but should prevent the internet from initiating an inbound IPv6 connection. The new architecture should also allow traffic flow inspection and traffic filtering.

What should a solutions architect do to meet these requirements?

- Launch the EC2 instance to a private subnet and attach AWS PrivateLink interface endpoint to the VPC to control outbound IPv6 communication to the internet. Use Amazon GuardDuty to set up the required rules for traffic inspection and traffic filtering.
- `Launch the EC2 instance to a private subnet and attach an Egress-Only Internet Gateway to the VPC to allow outbound IPv6 communication to the internet. Use AWS Network Firewall to set up the required rules for traffic inspection and traffic filtering.
`
- Launch the EC2 instance to a public subnet and attach an Internet Gateway to the VPC to allow outbound IPv6 communication to the internet. Use Traffic Mirroring to set up the required rules for traffic inspection and traffic filtering.
- Launch the EC2 instance to a private subnet and attach a NAT Gateway to the VPC to allow outbound IPv6 communication to the internet. Use AWS Firewall Manager to set up the required rules for traffic inspection and traffic filtering.

This is a **very AWS SAA-style networking question**. It looks difficult because it mentions **IPv6, Egress-Only Internet Gateway, Network Firewall, PrivateLink, GuardDuty, Firewall Manager, and Traffic Mirroring** all in one question.

The trick is to identify the **two separate requirements**:

1. **How should IPv6 traffic leave the VPC?**
2. **Which service inspects and filters that traffic?**

Let's solve it step by step.

---

# Step 1: Read the requirements

The question says:

> only allow **outbound** communication over IPv6

Keyword:

> **Outbound only**

---

Then:

> prevent the internet from initiating an inbound IPv6 connection

Keyword:

> **Block inbound**

---

Then:

> allow traffic flow inspection and traffic filtering

Keyword:

> **Firewall**

---

These are **three independent requirements**.

---

# Step 2: Why is IPv6 different?

With IPv4, most people know this architecture:

```text
Private EC2
      │
      ▼
NAT Gateway
      │
      ▼
Internet Gateway
      │
Internet
```

The EC2 instance can access the internet.

The internet **cannot start a connection back**.

---

Now think about IPv6.

IPv6 addresses are **globally unique**.

Example:

```text
2001:db8:1234::1
```

This is already a public address.

There is **no NAT** like traditional IPv4.

So AWS had to create another solution.

---

# Step 3: The Answer

AWS created:

> **Egress-Only Internet Gateway**

Remember the name.

**Egress**

means

> Outgoing traffic.

So:

```text
Private EC2

↓

Egress-Only Internet Gateway

↓

Internet
```

Allowed:

EC2 → Internet ✅

Blocked:

Internet → EC2 ❌

Exactly what the question asks.

---

# Step 4: Why not Internet Gateway?

Internet Gateway is bidirectional.

```text
EC2

↓

Internet Gateway

↓

Internet
```

Traffic:

EC2 → Internet ✅

Internet → EC2 ✅

(as long as Security Groups/NACLs allow it)

Question specifically says:

> prevent inbound IPv6

Therefore:

Internet Gateway is wrong.

---

# Step 5: Memory Trick

### Internet Gateway

Think:

Door.

People can enter.

People can leave.

---

### Egress-Only Internet Gateway

Think:

Emergency Exit.

You can go out.

Nobody can come back in.

That is exactly how it behaves.

---

# Step 6: What about Traffic Inspection?

Question also says:

> inspect traffic

and

> filter traffic

This is NOT the job of an Internet Gateway.

AWS service for that:

> **AWS Network Firewall**

Network Firewall is literally a firewall.

It can:

✅ Inspect packets

✅ Stateful inspection

✅ IPS

✅ Domain filtering

✅ Protocol filtering

✅ Threat detection

Exactly what the question asks.

---

# Step 7: Why Option A is wrong

It mentions:

PrivateLink

Many students see:

"Private"

and think it sounds secure.

But what is PrivateLink?

PrivateLink connects:

```text
Your VPC

↓

AWS Service

```

Examples:

* S3
* Secrets Manager
* SNS

without using the Internet.

It is **NOT** an internet gateway.

It cannot provide internet access.

---

Then it mentions:

GuardDuty.

GuardDuty:

Detects threats.

Example:

* Crypto mining

* Port scanning

* Suspicious API calls

It **doesn't block traffic**.

Think:

GuardDuty = Security Alarm.

Not Firewall.

---

# Step 8: Why Option C is wrong

It says:

Internet Gateway

Already wrong because:

Internet can initiate inbound connections.

---

Then:

Traffic Mirroring.

Traffic Mirroring:

Copies packets.

Imagine:

```text
EC2

↓

Copy packets

↓

Security appliance
```

It is like Wireshark.

Observation only.

Not filtering.

---

Think:

Traffic Mirroring = CCTV camera.

It records.

It doesn't stop criminals.

---

# Step 9: Why Option D is wrong

It says:

NAT Gateway.

Question:

Does NAT Gateway work for IPv6?

Traditional NAT Gateway is designed for IPv4. AWS supports **NAT64** in specific scenarios, but **it is not the mechanism used to provide outbound-only IPv6 internet access**.

AWS created a dedicated service:

> Egress-Only Internet Gateway

Whenever you see:

IPv6

Outbound only

Use:

Egress-Only Internet Gateway.

---

Then:

Firewall Manager.

Students confuse this constantly.

Firewall Manager does NOT inspect packets.

It manages firewall policies.

Imagine a company with:

200 AWS accounts.

Instead of configuring every firewall separately...

Firewall Manager pushes policies everywhere.

Think:

Manager.

Not firewall.

---

# Step 10: Compare the Services

## GuardDuty

Purpose:

Threat Detection.

Example:

"Hacker is scanning your EC2."

It alerts.

---

## Network Firewall

Purpose:

Traffic filtering.

Example:

"Block this packet."

---

## Firewall Manager

Purpose:

Central management.

Example:

"Apply this firewall rule to every AWS account."

---

## Traffic Mirroring

Purpose:

Packet capture.

Example:

"Copy traffic for analysis."

---

## PrivateLink

Purpose:

Private AWS service access.

Example:

EC2 → Secrets Manager

without Internet.

---

# Step 11: Architect View

Correct architecture:

```text
          Internet
              ▲
              │
   Egress-Only Internet Gateway
              ▲
              │
      AWS Network Firewall
              ▲
              │
        Private Subnet
              │
             EC2
```

Traffic:

EC2 → Internet ✅

Internet → EC2 ❌

Firewall inspects every packet.

---

# Step 12: SAA Memory Tricks

## IPv4

Private subnet

↓

NAT Gateway

↓

Internet

---

## IPv6

Private subnet

↓

Egress-Only Internet Gateway

↓

Internet

---

Notice:

IPv4 → NAT

IPv6 → Egress-Only IGW

This is one of the most tested networking facts.

# AWS Networking Cheat Sheet

| Requirement                                | AWS Service                      |
| ------------------------------------------ | -------------------------------- |
| Outbound IPv4 internet from private subnet | NAT Gateway                      |
| Outbound IPv6 internet from private subnet | **Egress-Only Internet Gateway** |
| Internet access both ways                  | Internet Gateway                 |
| Private access to AWS services             | VPC Endpoint / PrivateLink       |
| Detect suspicious activity                 | GuardDuty                        |
| Inspect and filter traffic                 | **AWS Network Firewall**         |
| Manage firewall policies across accounts   | Firewall Manager                 |
| Copy packets for analysis                  | Traffic Mirroring                |

---

## One exam shortcut

When you see **IPv6**, immediately slow down and ask yourself:

* Does the instance need **both inbound and outbound internet access**? → **Internet Gateway**
* Does it need **only outbound internet access**? → **Egress-Only Internet Gateway**

Then separately identify whether the question asks for **inspection/filtering** (→ **AWS Network Firewall**) or **threat detection** (→ **GuardDuty**). AWS often combines these in a single question to see if you can distinguish networking from security monitoring.


## 🎯 Summary
- **IPv4 private subnet → NAT Gateway** (translation + outbound only).  
- **IPv6 private subnet → Egress-Only Internet Gateway** (no translation, outbound only).  
- Both ensure private subnet instances can reach the internet securely, but inbound traffic is blocked.

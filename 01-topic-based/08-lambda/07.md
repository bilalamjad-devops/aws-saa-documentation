7. Question

Category: CSAA – Design High-Performing Architectures

A game development company operates several virtual reality (VR) and augmented reality (AR) games that use various RESTful web APIs hosted in its on-premises data center, which currently sits behind a content delivery network (CDN) for faster global delivery. Due to the unprecedented growth of the company, the management decided to migrate its system to AWS Cloud to scale out its resources as well as to minimize costs.

Which of the following is the most cost-effective and scalable solution to meet the above requirement?

- Host the APIs in a static Amazon S3 web hosting bucket behind a Amazon CloudFront web distribution.
- Set up a micro-service architecture with Amazon ECS, Amazon ECR, and AWS Fargate.
- `Use AWS Lambda and Amazon API Gateway.`
- Use a Spot Fleet of Amazon EC2 instances, each with an Elastic Fabric Adapter (EFA) for more consistent latency and higher network throughput. Set up an Application Load Balancer to distribute traffic to the instances.


## Question Summary

### Keywords

* RESTful Web APIs
* Migrating to AWS
* **Scalable**
* **Cost-effective**
* Dynamic APIs

---

# Correct Answer

✅ **Amazon API Gateway + AWS Lambda**

Architecture:

```text
Client
   │
   ▼
API Gateway
   │
   ▼
Lambda
```

---

# Why?

The application exposes **REST APIs**.

AWS provides a fully serverless solution:

* API Gateway receives HTTP requests.
* Lambda executes the backend code.
* No servers to manage.
* Automatic scaling.
* Pay only when requests arrive.

This is one of the most common serverless architectures in AWS.

---

# Why the Others Are Wrong

## ❌ ECS + ECR + Fargate

This is a valid architecture **if your application is containerized**.

Use when:

* Docker containers
* Long-running applications
* Microservices

But the scenario simply asks for **REST APIs** with the **most cost-effective** solution.

Serverless (API Gateway + Lambda) is cheaper and simpler.

---

## ❌ S3 + CloudFront

S3 can host only:

* Static HTML
* CSS
* JavaScript
* Images

It **cannot execute backend code** or REST APIs.

No compute capability.

---

## ❌ Spot Fleet + ALB + EFA

Problems:

* Still managing EC2 servers
* More operational overhead
* Spot instances can be interrupted
* EFA is unrelated

### EFA (Elastic Fabric Adapter)

Used for:

* HPC (High Performance Computing)
* Machine Learning
* MPI applications
* Low latency networking

Not for normal REST APIs.

---

# Exam Tips

## If you see

* REST API
* Serverless
* Cheapest
* Auto Scaling
* Event-driven

➡️ **API Gateway + Lambda**

---

## When to Use API Gateway

Acts as the front door for:

* REST APIs
* HTTP APIs
* WebSocket APIs

Routes requests to:

* Lambda
* ECS
* EC2
* Other HTTP endpoints

---

## Lambda vs ECS

Use **Lambda** when:

* Short-lived execution
* Event-driven
* REST APIs
* No infrastructure management

Use **ECS/Fargate** when:

* Docker containers
* Long-running services
* Stateful applications
* Custom runtimes

---

# Quick Comparison

| Requirement                  | Service           |
| ---------------------------- | ----------------- |
| REST API                     | ✅ API Gateway     |
| Backend compute              | ✅ Lambda          |
| Serverless                   | ✅ Lambda          |
| Containerized apps           | ✅ ECS + Fargate   |
| Static website               | ✅ S3 + CloudFront |
| HPC / Low-latency networking | ✅ EFA             |

8. Question

Category: CSAA – Design Resilient Architectures

A company is using Amazon VPC that has a CIDR block of 10.31.0.0/27 that is connected to the on-premises data center. There was a requirement to create a Lambda function that will process massive amounts of cryptocurrency transactions every minute and then store the results to EFS. After setting up the serverless architecture and connecting the Lambda function to the VPC, the Solutions Architect noticed an increase in invocation errors with EC2 error types such as EC2ThrottledException at certain times of the day.

Which of the following are the possible causes of this issue? (Select TWO.)

- The associated security group of your function does not allow outbound connections.
- `Your Lambda function exceeds the VPC quota for Elastic Network Interfaces (ENIs) or available IP addresses in the subnet.`
- Your VPC does not have a NAT gateway.
- `The Lambda function is placed in a VPC subnet with limited IP address capacity.`
- The attached IAM execution role of your function does not have the necessary permissions to access the resources of your VPC.


## Question Summary

### Keywords

* **Lambda inside a VPC**
* Stores data in **Amazon EFS**
* **EC2ThrottledException**
* Happens only during peak traffic
* **Select TWO**

---

# Correct Answers

✅ **Lambda exceeds ENI quota / available IP addresses**

✅ **Subnet has limited IP address capacity**

---

# Why?

When you connect **Lambda to a VPC**, AWS creates **Elastic Network Interfaces (ENIs)** so Lambda can communicate with VPC resources (like EFS, RDS, or EC2).

```text
Lambda
   │
   ▼
ENI
   │
   ▼
VPC
   │
   ▼
EFS
```

During traffic spikes, Lambda scales by creating more execution environments, which may require additional ENIs and subnet IP addresses.

If there aren't enough:

* Available IP addresses
* ENI quota

Lambda can't scale and starts returning:

> **EC2ThrottledException**

---

# Hidden Clue

The VPC CIDR is:

```text
10.31.0.0/27
```

A **/27 subnet** has only **32 total IP addresses** (about **27 usable** after AWS reserves some).

That's a **very small subnet**.

For a high-concurrency Lambda function, it can quickly run out of IPs.

---

# Why Others Are Wrong

## ❌ NAT Gateway

NAT is needed only when private resources need outbound Internet access.

Running out of NAT capacity does **not** produce:

> EC2ThrottledException

---

## ❌ Security Group

If outbound traffic were blocked:

* Lambda would fail **every time**, not only during peak hours.

---

## ❌ IAM Role

A missing IAM permission causes:

* `AccessDenied`
* `EC2AccessDeniedException`

Not:

* `EC2ThrottledException`

---

# Exam Tips

## Lambda + VPC

Remember:

Connecting Lambda to a VPC means Lambda needs:

* ENIs
* Private IP addresses

No VPC → No ENIs.

---

## Common VPC Resources Requiring Lambda in a VPC

* Amazon EFS ✅
* Amazon RDS ✅
* EC2 ✅
* ElastiCache ✅

---

## Common Errors

### EC2ThrottledException

Think:

* ENI limit reached
* No available subnet IPs

---

### EC2AccessDeniedException

Think:

* IAM permissions missing

---

### Timeout

Think:

* NAT
* Route table
* Security group
* Network connectivity

---

# Quick Comparison

| Error                    | Most Likely Cause                         |
| ------------------------ | ----------------------------------------- |
| EC2ThrottledException    | ✅ ENI limit / IP exhaustion               |
| EC2AccessDeniedException | ✅ IAM permissions                         |
| Timeout                  | ✅ Network configuration (NAT, routes, SG) |
| Lambda stops at 15 min   | ✅ Timeout limit                           |

# Exam Shortcut

| Keywords                    | Answer                            |
| --------------------------- | --------------------------------- |
| **Lambda + VPC**            | ENIs required                     |
| **Lambda + EFS**            | Must be in VPC                    |
| **EC2ThrottledException**   | ENI quota or subnet IP exhaustion |
| **Small CIDR (/27, /28)**   | Possible IP shortage              |
| **High Lambda concurrency** | More ENIs and IPs needed          |

### Memory Trick

When you see:

* **Lambda**
* **VPC**
* **EFS/RDS**
* **EC2ThrottledException**
* **Traffic spikes**

➡️ Immediately think:

**"Lambda can't create more ENIs because it has run out of ENI quota or subnet IP addresses."**

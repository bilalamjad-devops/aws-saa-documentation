8. Question

Category: CSAA – Design Cost-Optimized Architectures

A company has multiple AWS sandbox accounts that are used by its development team. All developers must be given access to the contents of one of the main account’s S3 buckets. For security purposes, any personally identifiable information (PII) or financial data uploaded in the bucket must be continuously monitored and removed.

How can this be done at the lowest possible cost and with the least amount of configuration effort?

- `Create an S3 bucket policy that grants access from the sandbox accounts. Use Amazon Macie to discover personally identifiable information (PII) or financial data.`
- Add S3 read permission to the IAM policy of each IAM user from the sandbox accounts. Use Amazon Detective to discover personally identifiable information (PII) or financial data.
- Configure cross-account replication on the S3 bucket. Integrate AWS Audit Manager with the S3 bucket to discover any personally identifiable information (PII) or financial data.
- Generate a pre-signed URL for the objects on the S3 bucket. Use the Amazon S3 Storage Lens to discover personally identifiable information (PII) or financial data.



This question tests **two separate concepts**:

1. **Cross-account S3 access**
2. **Finding sensitive data (PII/financial data)**

Split the question first, then solve each requirement.

---

# Requirement 1

> Developers from multiple AWS accounts need access to one S3 bucket.

Think:

**Cross-account access**

Possible solutions:

* Bucket Policy ✅
* IAM Role ✅
* ACL (old)
* IAM policy on every user ❌ (too much work)

Question says:

> **lowest cost**
>
> **least configuration**

Therefore

✅ **Bucket Policy**

**Keyword**

Cross-account + S3

→ Bucket Policy

---

# Requirement 2

> Continuously monitor and discover PII / financial information.

Only one AWS service does this.

✅ **Amazon Macie**

Macie scans S3 and finds:

* Personally Identifiable Information (PII)
* Credit cards
* Financial records
* Sensitive data

Think:

Sensitive Data in S3

→ **Macie**

---

# Why others are wrong

### IAM policy for every user

❌

300 users?

500 users?

1000 users?

Huge management overhead.

Bucket Policy is much easier.

---

### Cross-Account Replication

Replication means

Copy Object

Question never asked for copies.

Need

Access

NOT

Replication

---

### Audit Manager

Audit Manager

≠ scans S3 data

It helps with

Compliance reports

Evidence collection

Audits

NOT PII detection.

---

### Amazon Detective

Detective

Analyzes

* GuardDuty findings
* VPC Flow Logs
* CloudTrail

It investigates attacks.

It does NOT scan S3 files.

---

### Storage Lens

Storage Lens

Shows

* Storage usage
* Cost
* Object count
* Request metrics

It cannot inspect object contents.

---

### Presigned URL

Temporary object sharing.

Not suitable for

* Entire development team
* Long-term access

# IAM SAA Cheat Sheet (Exam Focus)

## 1. IAM Users vs Groups vs Roles

| Service       | Use Case                                                   |
| ------------- | ---------------------------------------------------------- |
| **IAM User**  | Permanent identity for one person/application              |
| **IAM Group** | Manage permissions for many IAM users together             |
| **IAM Role**  | Temporary access (AWS services, cross-account, federation) |

### Remember

* User → long-term identity
* Group → permission management
* Role → temporary credentials

---

# 2. IAM Group

Use when **many IAM users need the same permissions.**

Example

100 developers need S3 access.

❌ Attach policy to 100 users

✅ Create Group → Attach policy once → Add users

**Keyword**

> Many users → IAM Group

---

# 3. IAM Role

Use when someone or something **temporarily needs permissions.**

Examples

* EC2 accessing S3
* Lambda accessing DynamoDB
* Cross-account access
* Temporary developer access
* Federated users

**Keyword**

> Temporary Access = Role

---

# 11. GuardDuty vs CloudTrail vs Inspector

| Service        | Purpose                                                            |
| -------------- | ------------------------------------------------------------------ |
| **GuardDuty**  | Detect suspicious activity, malware, unusual API calls, S3 threats |
| **CloudTrail** | Records API calls for auditing                                     |
| **Inspector**  | Finds EC2/ECR/Lambda vulnerabilities                               |
| **AWS WAF**    | Blocks SQL Injection, XSS                                          |
| **Shield**     | DDoS protection                                                    |

Easy way to remember:

* **CloudTrail = Who did what?**
* **GuardDuty = Is someone attacking me?**
* **Inspector = Is my workload vulnerable?**
* **WAF = Block bad web requests**
* **Shield = Stop DDoS**

## 30-Second Decision Tree

* **Many users?** → IAM Group
* **Temporary access?** → IAM Role + STS
* **Cross-account?** → AssumeRole + STS
* **Active Directory?** → SAML Federation (ADFS)
* **LDAP, no SAML?** → Custom Identity Broker + STS
* **Programmatic access?** → Access Keys
* **S3 suspicious activity?** → GuardDuty
* **Need API audit logs?** → CloudTrail
* **Need to stop SQL Injection/XSS?** → WAF
* **Need vulnerability scanning?** → Inspector

1. Question

Category: CSAA – Design Secure Architectures

A company has an application that continually sends encrypted documents to Amazon S3. The company requires that the configuration for data access is in line with its strict compliance standards. It should also be alerted if there is any risk of unauthorized access or suspicious access patterns.

Which step is needed to meet the requirements?

- Use Amazon Inspector to alert whenever a security violation is detected on S3.
- Use AWS CloudTrail to monitor and detect access patterns on S3.
- `Use Amazon GuardDuty to monitor malicious activity on S3.`
- Use Amazon Rekognition to monitor and recognize patterns on S3.

Good start with IAM. This question is really testing the difference between **CloudTrail**, **GuardDuty**, and **Inspector**.

### Keywords in the question

* **Encrypted documents in S3** → just background.
* **Strict compliance** → background.
* **Alert if there is any risk of unauthorized access or suspicious access patterns** ← **this is the key.**

The words **"alert", "unauthorized access", "suspicious activity"** should immediately make you think of **GuardDuty**.

---

## Why GuardDuty? ✅

GuardDuty is an **intelligent threat detection service**.

It continuously analyzes:

* CloudTrail events
* VPC Flow Logs
* DNS Logs
* **S3 Data Events** (when enabled)
* EKS audit logs

Then it uses:

* Machine Learning
* Threat Intelligence
* Anomaly Detection

to detect suspicious behavior.

Examples:

* Someone accessing your S3 bucket from a known malicious IP
* Unusual download of large amounts of data
* Credential compromise
* Reconnaissance activity
* Public bucket policy changes
* Suspicious API calls

Then it creates **Security Findings**.

Think:

> **GuardDuty = Detect + Alert**

---

## Why not CloudTrail?

CloudTrail simply **records** API calls.

Example:

```
User Alice
↓

GetObject
↓

Bucket: finance-data
↓

Time: 10:45
```

CloudTrail says:

> "This API call happened."

It **doesn't say**

> "This looks suspicious."

You would need to analyze the logs yourself.

Think:

> **CloudTrail = CCTV Camera**

It records everything.

It doesn't call the police.

---

## Why not Inspector?

Inspector scans:

* EC2
* ECR Images
* Lambda

It finds:

* CVEs
* Vulnerabilities
* Missing patches

It does **NOT** monitor S3 access.

Think:

> Inspector = Vulnerability Scanner

Not intrusion detection.

---

## Why not Rekognition?

Rekognition analyzes:

* Images
* Videos
* Faces
* Text

Nothing related to S3 security.

---

# Easy memory trick

| Service     | Think                                       |
| ----------- | ------------------------------------------- |
| CloudTrail  | Who did what? (Audit Logs)                  |
| GuardDuty   | Is something suspicious? (Threat Detection) |
| Inspector   | Is my EC2/Lambda/ECR vulnerable?            |
| Rekognition | Image & Video AI                            |

---

# SAA Exam Shortcut

Whenever you see:

* suspicious activity
* malicious activity
* compromised credentials
* unusual API calls
* known malicious IP
* threat detection
* anomaly detection

👉 **Answer = GuardDuty**

Whenever you see:

* audit
* API history
* compliance logs
* who deleted bucket
* who created IAM user

👉 **Answer = CloudTrail**

Whenever you see:

* vulnerabilities
* CVEs
* patch management
* package scanning
* EC2/Lambda/ECR security assessment

👉 **Answer = Inspector**

---

### One-line exam rule

* **CloudTrail = Records events**
* **GuardDuty = Detects threats and raises findings**
* **Inspector = Finds vulnerabilities**
* **Rekognition = Image/video analysis**

7. Question

Category: CSAA – Design Secure Architectures

An online events registration system is hosted in AWS and uses ECS to host its front-end tier and an RDS configured with Multi-AZ for its database tier. What are the events that will make Amazon RDS automatically perform a failover to the standby replica? (Select TWO.)

- `Loss of availability in primary Availability Zone`
- Compute unit failure on secondary DB instance
- In the event of Read Replica failure
- Storage failure on secondary DB instance
- `Storage failure on primary`

This is another question where AWS tries to confuse you. The trick is actually very simple.

# One Rule

**Multi-AZ exists to protect the PRIMARY database.**

Ask yourself:

> **Did something happen to the PRIMARY?**

* Yes → Failover
* No → No failover

---

## Architecture

```
Users
   |
Primary DB  (AZ-1)
      ||
      || Synchronous Replication
      ||
Standby DB (AZ-2)
```

The standby just waits.

It is **not** used for reads.

---

## When does failover happen?

### ✅ Storage failure on Primary

```
Primary Storage crashes
      ↓
Standby becomes Primary
```

✔️ Automatic failover

---

### ✅ Loss of Availability Zone (Primary)

```
AZ-1 goes down
      ↓
Primary unavailable
      ↓
Standby promoted
```

✔️ Automatic failover

---

## When does failover NOT happen?

### ❌ Read Replica failure

Read Replicas are completely different from Multi-AZ.

```
Primary
   |
Read Replica
```

If the replica dies:

* Reads from that replica fail.
* Primary is still healthy.
* No failover.

---

### ❌ Storage failure on Secondary

```
Primary  ✅

Standby ❌
```

Primary is still serving traffic.

AWS repairs or recreates the standby.

No failover.

---

### ❌ Compute failure on Secondary

Same idea.

```
Primary  ✅

Standby ❌
```

Primary keeps running.

No failover.

---

# Easy Exam Trick

Think of the standby as a **backup player** in a football team.

If the **captain (Primary)** gets injured:

➡️ Backup player comes in.

If the **backup player** gets injured:

➡️ Captain keeps playing.

No substitution is needed.

---

# Memorize This

## Multi-AZ Failover happens if the **Primary** has:

* ✅ Storage failure
* ✅ Compute failure
* ✅ Network failure
* ✅ Availability Zone failure

---

## No failover if only the **Standby** has problems:

* ❌ Standby storage failure
* ❌ Standby compute failure
* ❌ Read Replica failure

---

## 5-Second Rule for the Exam

Whenever you see a Multi-AZ question, ask:

> **"Is the PRIMARY affected?"**

* **Yes** → Automatic failover.
* **No** → No automatic failover.

This single rule will answer almost every Multi-AZ failover question correctly.

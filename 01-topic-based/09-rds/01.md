An application that records weather data every minute is deployed in a fleet of Amazon EC2 Spot instances and uses a MySQL RDS database instance. Currently, there is only one Amazon RDS instance running in one Availability Zone. The database needs to be improved to ensure high availability by enabling synchronous data replication to another RDS instance.

Which of the following performs synchronous data replication in RDS?

- Amazon CloudFront running as a Multi-AZ deployment
- RDS Read Replica
- `RDS DB instance running as a Multi-AZ deployment`

# Important Concepts

## 1. RDS Multi-AZ = High Availability

Purpose:

* Protect against AZ failure
* Automatic failover
* Disaster recovery

Replication:

* **Synchronous**

Architecture:

```
Primary DB (AZ-A)
        │
        │  Synchronous Replication
        ▼
Standby DB (AZ-B)
```

If Primary fails:

```
Primary ❌

↓

Standby becomes Primary automatically
```

Application reconnects automatically.

---

## 2. Read Replica = Read Scaling

Purpose:

* Increase read performance
* Reporting
* Analytics

Replication:

* **Asynchronous**

Architecture

```
Primary
      │
      │ Async Replication
      ▼
Read Replica
```

Can have multiple replicas.

Used for:

* Heavy SELECT queries
* Reporting
* BI dashboards

NOT for HA.

# Synchronous vs Asynchronous

### Multi-AZ

```
Application

↓

Primary writes

↓

Standby also writes

↓

Only then success returned
```

Pros

* No data loss

Cons

* Slightly slower writes

---

### Read Replica

```
Primary writes

↓

Success returned immediately

↓

Replica updated later
```

Pros

* Fast

Cons

* Small replication lag

# Wrong Options

### ❌ RDS Read Replica

Wrong because:

* Replication is **asynchronous**
* Used for **read scaling**, not HA.

---

### ❌ CloudFront

CloudFront is a CDN.

It caches web content at edge locations.

Nothing to do with databases.

---

### ❌ DynamoDB Read Replica

DynamoDB doesn't use "Read Replicas."

For cross-Region replication it uses:

* **Global Tables**

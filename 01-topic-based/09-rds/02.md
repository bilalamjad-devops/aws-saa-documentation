# Correct Answer 1

## Force SSL

```
rds.force_ssl = true
```

This tells RDS:

> "I refuse any non-encrypted connection."

Without SSL:

```
EC2 -------> RDS
```

Anyone sniffing traffic could read it.

With SSL:

```
EC2 ==Encrypted==> RDS
```

Nobody can read it.

✅ Correct

## ❌ IAM Database Authentication

IAM Authentication replaces passwords.

Instead of:

```
Username
Password
```

You use an IAM token.

It improves authentication.

It **does not encrypt traffic**.

Question asks:

Secure data while traveling.

Not:

Secure login.

Wrong.

---

## ❌ Security Group Port 443

Many students choose this.

Why is it wrong?

Security Groups only allow or deny traffic.

Think of a security guard.

```
Can enter?

YES

Can enter?

NO
```

The guard does NOT encrypt your conversation.

Even if you allow only port 443:

```
EC2 --------> RDS
```

Nothing guarantees SSL unless SSL is enabled.

Security Group controls access.

SSL encrypts data.

Different jobs.

## One-Line Memory

* **Data moving** → **SSL/TLS**
* **Data stored** → **TDE/KMS**
* **Who can connect?** → **Security Group**
* **Who is connecting?** → **IAM Authentication**

This simple distinction ("moving vs stored") will help you answer many RDS security questions correctly.

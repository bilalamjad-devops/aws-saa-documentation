3. Question

Category: CSAA – Design Secure Architectures

A financial application consists of an Auto Scaling group of Amazon EC2 instances, an Application Load Balancer, and a MySQL RDS instance set up in a Multi-AZ Deployment configuration. To protect customers’ confidential data, it must be ensured that the Amazon RDS database is only accessible using an authentication token specific to the profile credentials of EC2 instances.

Which of the following actions should be taken to meet this requirement?

- Create an IAM Role and assign it to your EC2 instances which will grant exclusive access to your RDS instance.
- Configure SSL in your application to encrypt the database connection to RDS.
- `Enable the IAM DB Authentication.`
- Use a combination of IAM and STS to enforce restricted access to your RDS instance using a temporary authentication token.

This is actually **much easier** than the previous question. The exam is trying to distract you.

---

# The Requirement

> **Database is only accessible using an authentication token**

Stop.

Circle this:

> **authentication token**

Now ask:

**Which AWS service gives authentication tokens for RDS?**

Answer:

> **IAM DB Authentication**

Done.

# Think Like This

Normally you connect to MySQL like this:

```text
Username: admin
Password: MyPassword123
```

Password is stored somewhere.

---

AWS says:

"No password."

Instead:

```text
Username: admin

Authentication Token
```

The token expires after **15 minutes**.

Much safer.

# Why Other Answers Are Wrong

## ❌ IAM Role

Many students pick this.

Why?

Because EC2 needs an IAM Role.

True.

But the question asks:

**How does RDS authenticate?**

Not

**How does EC2 get permissions?**

The role lets EC2 **request** the token.

But RDS must also support IAM authentication.

So the feature you need is:

✅ IAM DB Authentication.

---

## ❌ SSL

SSL only does this:

```text
EC2 =====encrypted=====> RDS
```

It protects the data while moving.

It does NOT replace the password.

Question asks:

Authentication token.

Wrong answer.

---

## ❌ STS

STS creates temporary AWS credentials.

Example:

```text
AWS Access Key

Secret Key

Session Token
```

These are for AWS services.

But connecting to MySQL RDS is a special feature called:

**IAM DB Authentication**.

The exam wants the specific service, not the generic token service.

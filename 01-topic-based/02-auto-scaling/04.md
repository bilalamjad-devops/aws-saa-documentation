4. Question

Category: CSAA – Design Resilient Architectures

A suite of web applications is hosted in an Auto Scaling group of Amazon EC2 instances across three Availability Zones and is configured with default settings. There is an Application Load Balancer that forwards the request to the respective target group on the URL path. The scale-in policy has been triggered due to the low number of incoming traffic to the application.

Which EC2 instance will be the first one to be terminated by the Auto Scaling group?

- `The EC2 instance launched from the oldest launch template.`
- The EC2 instance which has been running for the longest time
- The instance will be randomly selected by the Auto Scaling group
- The EC2 instance which has the least number of user sessions


Excellent question. This is one of the **hardest ASG questions** because it tests something many engineers never use in day-to-day work:

> **Default ASG Termination Policy**

The good news?

For the SAA exam, you only need to remember the **logic**, not every tiny detail.

---

# Step 1: What is Scale-In?

You already know:

Scale-Out

```text
4 EC2

↓

Traffic increases

↓

6 EC2
```

---

Scale-In

```text
6 EC2

↓

Traffic decreases

↓

4 EC2
```

Question:

> Which EC2 should AWS terminate?

That's what this question is asking.

---

# Step 2: The Important Part

The question says:

> Auto Scaling group with **default settings**

Those two words are the clue.

Default settings mean:

AWS uses its **Default Termination Policy**.

---

# Step 3: AWS doesn't randomly kill instances

Imagine this architecture:

```text
AZ-A

EC2
EC2
EC2

AZ-B

EC2
EC2

AZ-C

EC2
```

Total:

6 instances

Traffic decreases.

Need:

5 instances.

Which one should go?

AWS doesn't randomly choose.

It first asks:

> Which Availability Zone has **more instances** than the others?

Answer:

AZ-A.

AWS wants to keep the AZs balanced.


Balanced.

# Step 4: Inside that AZ

Suppose AZ-A contains:

```text
Instance 1

Launch Template V1
```

```text
Instance 2

Launch Template V2
```

```text
Instance 3

Launch Template V2
```

Which one gets terminated?

AWS prefers removing:

**The instance using the oldest launch template.**

Why?

Because AWS wants your fleet to gradually move toward the newest launch template.

---

# Step 5: Why?

Suppose Launch Template V1 has:

* Old AMI
* Old security patches
* Old software

Launch Template V2 has:

* Latest AMI
* Latest patches
* Latest application

During scale-in,

AWS removes the older generation first.

This helps keep the fleet modern.

---

# Step 6: Option Analysis

### Option A ✅

> EC2 launched from the oldest Launch Template

Correct.

Exactly how the default termination policy works after selecting the appropriate AZ.

---

### Option B

> Longest running instance

Sounds reasonable.

But AWS doesn't care about uptime.

Wrong.

---

### Option C

> Random

Only used as a **last tie-breaker** after other rules are exhausted.

Not the default first choice.

Wrong.

---

### Option D

> Least user sessions

AWS Auto Scaling doesn't inspect application sessions when deciding which instance to terminate.

Wrong.

---

# Step 7: The Real Order

AWS follows this sequence.

## Step 1

Choose the Availability Zone with:

Most instances.

Because AWS wants balanced AZs.

---

## Step 2

Inside that AZ,

Find instances using:

Oldest Launch Template.

---

## Step 3

If multiple candidates remain,

Historically, AWS considered the instance **closest to the next billing hour** to maximize value from hourly billing. Today, EC2 On-Demand instances are billed per second for Linux (with a 60-second minimum), so this criterion is far less important than it once was, but you'll still see it in some exam materials because it's part of the documented termination policy flow.

---

## Step 4

Still tied?

Choose randomly.

# Step 9: Memory Trick

Imagine a company has employees.

Some still use:

Windows XP laptops.

# Step 10: Exam Shortcut

Whenever you see:

> Default Termination Policy

Immediately think:

> **Balance Availability Zones first, then terminate an instance using the oldest Launch Template.**

You don't need to memorize every tie-breaker unless the exam explicitly asks.

## ⭐ Bilal's Golden Rule #14

When an Auto Scaling Group scales **in**, AWS tries to:

1. **Keep Availability Zones balanced.**
2. **Prefer terminating instances launched from the oldest Launch Template.**
3. **Use additional tie-breakers only if needed.**

For the SAA exam, if you see **"default termination policy"**, the safest answer is usually the one involving **balancing Availability Zones and removing the oldest Launch Template instance**.

5. Question

Category: CSAA – Design Resilient Architectures

A tech company is currently using Auto Scaling for their web application. A new AMI now needs to be used for launching a fleet of EC2 instances. Which of the following changes needs to be done?

- `Create a new launch template.`
- Create a new target group and launch template.
- Do nothing. You can start directly launching EC2 instances in the Auto Scaling group with the same launch template.
- Create a new target group.



Excellent. This is one of the **highest-yield ASG questions**. The exam loves asking about **Launch Templates** because every Auto Scaling Group depends on one.

---

# Step 1: First understand the relationship

Think of an Auto Scaling Group like a factory.

The factory (ASG) creates new EC2 instances.

But how does it know **what kind** of EC2 to create?

It needs a blueprint.

That blueprint is called a:

> **Launch Template**

---

Architecture:

```text
Auto Scaling Group
        │
        ▼
 Launch Template
        │
        ▼
Creates EC2 Instances
```

Without a Launch Template,

ASG has no idea:

* Which AMI?
* Which Instance Type?
* Which Security Group?
* Which Key Pair?
* Which IAM Role?
* Which User Data?

Everything comes from the Launch Template.

---

# Step 2: What's inside a Launch Template?

A Launch Template stores the configuration for new EC2 instances.

Typical contents:

```text
AMI

Instance Type

Security Groups

IAM Role

Key Pair

Storage (EBS)

User Data Script

Network Settings

Tags
```

Think of it as:

> **A recipe for creating EC2 instances.**

---

# Step 3: The question

The company wants to use:

> **A new AMI**

Question:

Can you edit the existing Launch Template?

Answer:

❌ No.

Launch Templates are **immutable**.

Once created,

You cannot edit them.

Instead,

You create:

A new version or a new Launch Template, then update the ASG to use it.

For the SAA exam, Tutorials Dojo simplifies this as:

> **Create a new Launch Template.**

---

# Step 4: Why?

Imagine your Launch Template looks like this:

```text
Launch Template

AMI = Amazon Linux 2023 v1

Instance = t3.micro
```

Now AWS releases:

Amazon Linux 2023 v2

Can you change:

```text
AMI = v2
```

inside the existing template?

No.

Create a new Launch Template (or a new version of the Launch Template) and tell the ASG to use it.

---

# Step 5: Option Analysis

## Option A ✅

Create a new Launch Template.

Correct.

---

## Option B

New Target Group + Launch Template

Wrong.

Target Groups belong to:

ALB

NLB

They have nothing to do with changing an AMI.

---

## Option C

Do nothing.

Wrong.

The existing Launch Template still references:

Old AMI.

Every new EC2 would continue using that old AMI.

---

## Option D

Create Target Group

Target Group only decides:

Where the Load Balancer sends traffic.

It doesn't define how EC2 instances are launched.

Wrong.

---

# Step 6: Launch Template vs Target Group

Students often confuse these.

## Launch Template

Creates EC2.

Contains:

```text
AMI

Instance Type

Security Group
```

Think:

**How to build an EC2.**

---

## Target Group

Used by Load Balancers.

Contains:

```text
EC2

Lambda

IP Addresses
```

Think:

**Where to send traffic.**

---

# Step 7: Real Architecture

```text
Internet

↓

ALB

↓

Target Group

↓

Auto Scaling Group

↓

Launch Template

↓

EC2
```

Notice:

Target Group

↓

Routes traffic.

Launch Template

↓

Creates EC2.

Different responsibilities.

# Step 8: Real DevOps Workflow

Suppose today your application uses:

Amazon Linux 2023

Tomorrow:

New security patches arrive.

You create:

Launch Template Version 2

↓

Update ASG

↓

Perform an Instance Refresh (or otherwise replace instances)

↓

New instances launch with the new AMI.

This is how production environments are updated.

---

# Step 9: Memory Trick

Imagine you're building houses.

Launch Template

↓

House Blueprint

If you want a new design...

Do you edit every existing house?

No.

You create:

A new blueprint.

Future houses use the new blueprint.

Exactly how Launch Templates work.

# ⭐ Bilal's Golden Rule #15

Whenever an SAA question asks:

> **"We need to launch future EC2 instances with a new AMI (or new instance configuration)."**

Think:

> **Update the Launch Template used by the Auto Scaling Group** (commonly by creating a new launch template or a new launch template version, then pointing the ASG to it).

---

## One exam note

The Tutorials Dojo explanation says **"Create a new launch template."** That's correct for the exam. In real AWS, **Launch Templates support versioning**, so you will often create a **new version** of an existing Launch Template and update the ASG to use that version rather than creating an entirely separate Launch Template. For SAA questions, the key concept is the same:

* ❌ You **cannot directly modify** an existing Launch Template.
* ✅ You must use a **new template/version** and update the ASG accordingly.

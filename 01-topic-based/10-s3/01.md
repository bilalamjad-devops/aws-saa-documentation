
1. Question

Category: CSAA – Design Secure Architectures

For data privacy, a healthcare company has been asked to comply with the Health Insurance Portability and Accountability Act (HIPAA). The company stores all its backups on an Amazon S3 bucket. It is required that data stored on the S3 bucket must be encrypted.

What is the best option to do this? (Select TWO.)

- `Enable Server-Side Encryption on an S3 bucket to make use of AES-256 encryption.`
- Store the data on Amazon EBS volumes with encryption enabled instead of using S3.
- Store the data in encrypted Amazon EBS snapshots.
- Enable Server-Side Encryption on an S3 bucket to make use of AES-128 encryption.
- `Before sending the data to S3 over HTTPS, encrypt the data locally first using your own encryption keys.`

Excellent! This is the **first S3 question**, and it introduces one of the **most important S3 topics: Encryption**.

---

# Step 1: What is the requirement?

The question says:

> **Data stored on S3 must be encrypted.**

Keyword:

```text
Stored
```

This means

> **Encryption at Rest**

Not while traveling over the network.

---

# Solution 1 (AWS encrypts it)

Enable

✅ **Server-Side Encryption (SSE)**

AWS encrypts every object before saving it.

```text
Your File
     |
     v
AWS encrypts it
     |
     v
Stored in S3
```

For S3, the encryption algorithm is:

✅ **AES-256**

Remember:

> **S3 Server-Side Encryption = AES-256**

---

# Solution 2 (You encrypt it)

Instead of AWS encrypting,

you encrypt the file first.

```text
Your Computer

↓

Encrypt File

↓

Upload to S3
```

AWS stores the encrypted file.

This is called:

**Client-Side Encryption**

You manage your own encryption keys.

---

# Why are BOTH answers correct?

AWS gives you two valid ways to encrypt S3 data.

### Option 1

AWS encrypts.

```text
SSE-S3
```

---

### Option 2

You encrypt.

```text
Client-Side Encryption
```

Both satisfy

> Data must be encrypted.

---

# Why are the other options wrong?

## ❌ EBS Volume Encryption

Question is about

```text
S3
```

EBS encryption only works for

```text
EC2 Disk
```

Different service.

---

## ❌ EBS Snapshot Encryption

Again

Snapshots belong to

```text
EBS
```

Not S3 encryption.

Wrong service.

---

## ❌ AES-128

S3 uses

```text
AES-256
```

Always remember:

> S3 Server-Side Encryption = **AES-256**

---

## One small update to your summary

The explanation mentions **three server-side encryption types**. For the SAA exam, remember these:

| Type        | Who manages the key? | Easy memory                                      |
| ----------- | -------------------- | ------------------------------------------------ |
| **SSE-S3**  | Amazon S3            | AWS-managed key (simplest)                       |
| **SSE-KMS** | AWS KMS              | AWS KMS key with more control and auditing       |
| **SSE-C**   | Customer             | You provide the encryption key with each request |

For now, just remember:

* **SSE-S3 → AWS-managed keys → AES-256**
* **SSE-KMS → AWS KMS keys**
* **SSE-C → Customer provides the key**

We'll cover the differences in more detail when encryption-specific S3 questions come up. This is another very common SAA topic.

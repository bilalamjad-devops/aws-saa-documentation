3. Question

Category: CSAA – Design Cost-Optimized Architectures

A start-up company that offers an intuitive financial data analytics service has consulted about its AWS architecture. The company has a fleet of Amazon EC2 worker instances that process financial data and then output reports used by its clients. The reports must be stored in a durable storage. The number of files to be stored can grow over time as the start-up company is expanding rapidly overseas, and thus, it also needs a way to distribute the reports faster to clients located across the globe. Although Amazon S3 One Zone-IA saves storage cost, it does not support global distribution.

Which of the following is a cost-efficient and scalable storage option for this scenario?

- Use S3 Glacier as the data storage and Amazon ElastiCache as the CDN.
- Use multiple EC2 instance stores for data storage and Amazon ElastiCache as the CDN.
- Use Amazon Redshift as the data storage and Amazon CloudFront as the CDN.
- `Use S3 as the data storage and Amazon CloudFront as the CDN.`

Excellent. S3 questions are much easier than RDS once you know the storage classes. Let's continue with the same simple style.

# Question 1 — S3 Encryption

### Keywords

* **HIPAA** → Secure data
* **S3 bucket**
* **Data must be encrypted**
* **Select TWO**

---

## Think like this

Question asks:

> How can we encrypt data in S3?

There are only two ways:

### Option 1 (AWS encrypts it)

**Server-Side Encryption (SSE)**

AWS encrypts after receiving the object.

Example:

```
Laptop
    |
HTTPS
    |
S3
(encrypts automatically)
```

SSE-S3 uses **AES-256**

✅ Correct

---

### Option 2 (You encrypt)

Encrypt before uploading.

```
Laptop
Encrypt
    |
HTTPS
    |
S3
```

This is called **Client-Side Encryption**

You manage your own keys.

✅ Correct

---

## Wrong Options

### EBS Encryption

Different service.

Question is about S3.

❌

---

### EBS Snapshots

Again EBS.

Not S3.

❌

---

### AES-128

S3 server-side encryption uses

**AES-256**

not AES-128.

❌

---

# Exam Trick

Whenever you see

> Encrypt S3 objects

Think

✅ SSE-S3 (AES-256)

or

✅ Client-side encryption

---

# Memory

```
S3 Encryption

Client encrypts
        OR
AWS encrypts (SSE)

SSE-S3 = AES-256
```

# Question 2 — S3 Standard-IA

Keywords:

* Not frequently accessed
* Immediately available
* Save money

---

## What is Standard-IA?

Imagine:

You have old company reports.

Nobody opens them every day.

But if the auditor comes,

he wants them

**NOW**

Not after hours.

That's Standard-IA.

---

### Characteristics

✅ Less frequently accessed

✅ Immediate access

✅ Lower storage cost

❌ Retrieval fee

---

## Correct answers

### Designed for less frequent access

Exactly what IA means.

✅

---

### Requires rapid access

Immediate retrieval.

No waiting.

✅

---

## Wrong answers

### Data Archiving

Archive = Glacier

❌

---

### Automatically moves data

That's

**Intelligent-Tiering**

❌

---

### High latency

No.

Standard-IA still provides

* Low latency
* High throughput

❌

---

# Easy Memory

```
Standard
↓
Standard-IA
↓
Glacier
↓
Deep Archive
```

Think:

Standard-IA =

> "I don't open it often,
> but when I do,
> I need it immediately."

# Question 3 — S3 + CloudFront

Keywords:

* Durable storage
* Millions of files
* Global users
* Faster downloads

---

## Break it

Need storage

→ S3

Need worldwide delivery

→ CloudFront

Done.

---

## Why not others?

### Glacier

Archive only.

Slow retrieval.

❌

---

### EC2 Instance Store

Temporary.

Instance dies

Data dies.

❌

---

### Redshift

Data warehouse.

Not for storing reports/files.

❌

---

## Correct Architecture

```
EC2 Workers

      ↓

      S3
(Store reports)

      ↓

CloudFront
(Global fast delivery)

      ↓

Clients
```

5. Question

Category: CSAA – Design Cost-Optimized Architectures

A few easily reproducible files must be stored in AWS without concerns about storage capacity. During the first month, all files will be accessed frequently, but after that period, access will become infrequent. Older files will only be accessed by developers, and no specific retrieval time requirement is defined. However, files under a specific <code>tdojo-finance</code> prefix in the Amazon S3 bucket will be used for post-processing that requires millisecond retrieval time.

Given these conditions, which of the following options would be the most cost-effective solution for the storage requirements?

- Store the files in S3, then after a month, change the storage class of the bucket to S3 Intelligent-Tiering using a lifecycle policy.
- Store the files in S3, then after a month, change the storage class of the bucket to S3 Standard-Infrequent Access (Standard-IA) using a lifecycle policy.
- Store the files in S3, then after a month, change the storage class of the tdojo-finance prefix to S3 Standard-Infrequent Access (Standard-IA) while the remaining go to Glacier using a lifecycle policy.
- `Store the files in S3, then after a month, change the storage class of the tdojo-finance prefix to S3 One Zone-Infrequent Access (One-Zone IA), while the remaining go to Glacier using a lifecycle policy.`

This is a **storage class question**. Don't read the long story—just extract the keywords.

# Step 1: Find the keywords

### Files are:

* ✅ Frequently accessed for 1 month
* ✅ Then infrequently accessed
* ✅ Easily reproducible
* ✅ Some files need millisecond retrieval
* ✅ Lowest cost

These keywords tell you everything.

---

# Step 2: Break the data into two groups

## Group 1

**Files under `tdojo-finance` prefix**

Need:

* Millisecond retrieval
* Rarely accessed

Cannot use Glacier because Glacier takes minutes or hours to retrieve.

Need:

**One Zone-IA**

Why?

* Immediate retrieval
* Cheapest IA storage
* Files are reproducible

---

## Group 2

Remaining files

Requirements:

* Rarely accessed
* No retrieval time requirement

Perfect for

**Glacier**

Very cheap.

# Why One Zone-IA?

AWS has two IA classes.

### Standard-IA

```text
2 Availability Zones
More durable
More expensive
```

---

### One Zone-IA

```text
1 Availability Zone
Cheaper
Lose an AZ → lose data
```

When do we use it?

AWS exam keyword:

> **Easily reproducible**

If data can be recreated,

use

**One Zone-IA**

to save money.

# Storage Class Cheat Sheet

| Requirement                                           | Storage Class        |
| ----------------------------------------------------- | -------------------- |
| Frequently accessed                                   | Standard             |
| Rarely accessed, instant retrieval                    | Standard-IA          |
| Rarely accessed, instant retrieval, reproducible data | **One Zone-IA**      |
| Unknown access pattern                                | Intelligent-Tiering  |
| Archive, retrieval can wait                           | Glacier              |
| Very long-term archive (cheapest)                     | Glacier Deep Archive |

## 10-second exam memory

```text
Frequently used?
→ Standard

Rarely used?
→ Standard-IA

Rarely + Easily Reproducible?
→ One Zone-IA

Unknown usage?
→ Intelligent-Tiering

Archive?
→ Glacier

Very old archive?
→ Deep Archive
```

This storage-class decision tree appears repeatedly in the SAA exam, so memorizing these keywords will help you answer many S3 questions quickly.

Very good question. This is the **trick** of the question.

The question is **not saying every file has the same requirement**.

It says:

> **Files under the `tdojo-finance` prefix** need millisecond retrieval.

That means only **one folder** has different requirements.

Imagine your bucket looks like this:

```text
my-bucket/

├── tdojo-finance/
│      report1.pdf
│      report2.pdf
│      report3.pdf
│
├── images/
│      img1.png
│      img2.png
│
├── logs/
│      log1.txt
│      log2.txt
│
└── backups/
       backup1.zip
```

Here:

* `tdojo-finance/` = special folder
* Everything else = normal files

---

## Why use the prefix?

Because those files are used for **post-processing**.

The question says:

> **requires millisecond retrieval time**

That means those files must open **immediately**.

So we **cannot** move them to Glacier.

Instead:

```text
tdojo-finance/
        ↓
One Zone-IA
```

because One Zone-IA still gives **instant (milliseconds)** access.

---

The other folders don't need fast retrieval.

So they can go to Glacier.

```text
logs/
backups/
images/

↓

Glacier
```

This saves much more money.

---

## Why not move the whole bucket to Glacier?

Suppose we move everything.

Later the application needs:

```text
tdojo-finance/report1.pdf
```

Oops...

It's in Glacier.

Retrieval could take:

* Minutes
* Hours

Application fails because it expected milliseconds.

---

## Why not move the whole bucket to One Zone-IA?

That also works.

But the question asks:

> **MOST cost-effective**

Glacier is **much cheaper** than One Zone-IA.

So AWS says:

* Only keep the files that need instant access in One Zone-IA.
* Archive everything else in Glacier.

---

## Think of it like a company

Imagine an office.

### Frequently used files

Keep on your desk.

(**S3 Standard**)

---

### Files you rarely use but sometimes need immediately

Keep in the office cupboard.

(**One Zone-IA**)

---

### Files you almost never use

Send them to the warehouse.

(**Glacier**)

# Exam Rule

Whenever you see:

* **prefix**
* **folder**
* **logs/**
* **images/**
* **finance/**

Think:

> AWS Lifecycle Policies can target **specific prefixes (folders)**.

For example:

```text
Rule 1:
Prefix = tdojo-finance/
→ Move to One Zone-IA after 30 days

Rule 2:
Prefix = *
→ Move to Glacier after 30 days
```

This is exactly why the question mentioned **`tdojo-finance`**—it wanted you to realize that **different folders can have different lifecycle rules and storage classes**. This is a very common SAA exam pattern.

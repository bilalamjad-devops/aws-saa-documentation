5. Question

Category: CSAA – Design Cost-Optimized Architectures

A few easily reproducible files must be stored in AWS without concerns about storage capacity. During the first month, all files will be accessed frequently, but after that period, access will become infrequent. Older files will only be accessed by developers, and no specific retrieval time requirement is defined. However, files under a specific <code>tdojo-finance</code> prefix in the Amazon S3 bucket will be used for post-processing that requires millisecond retrieval time.

Given these conditions, which of the following options would be the most cost-effective solution for the storage requirements?

- Store the files in S3, then after a month, change the storage class of the bucket to S3 Intelligent-Tiering using a lifecycle policy.
- Store the files in S3, then after a month, change the storage class of the bucket to S3 Standard-Infrequent Access (Standard-IA) using a lifecycle policy.
- Store the files in S3, then after a month, change the storage class of the tdojo-finance prefix to S3 Standard-Infrequent Access (Standard-IA) while the remaining go to Glacier using a lifecycle policy.
- `Store the files in S3, then after a month, change the storage class of the tdojo-finance prefix to S3 One Zone-Infrequent Access (One-Zone IA), while the remaining go to Glacier using a lifecycle policy.`

This is a **storage class question**. Don't read the long story—just extract the keywords.

---

# Step 1: Find the keywords

### Files are:

* ✅ Frequently accessed for 1 month
* ✅ Then infrequently accessed
* ✅ Easily reproducible
* ✅ Some files need millisecond retrieval
* ✅ Lowest cost

These keywords tell you everything.

---

# Step 2: Break the data into two groups

## Group 1

**Files under `tdojo-finance` prefix**

Need:

* Millisecond retrieval
* Rarely accessed

Cannot use Glacier because Glacier takes minutes or hours to retrieve.

Need:

**One Zone-IA**

Why?

* Immediate retrieval
* Cheapest IA storage
* Files are reproducible

---

## Group 2

Remaining files

Requirements:

* Rarely accessed
* No retrieval time requirement

Perfect for

**Glacier**

Very cheap.

---

# Why One Zone-IA?

AWS has two IA classes.

### Standard-IA

```text
2 Availability Zones
More durable
More expensive
```

---

### One Zone-IA

```text
1 Availability Zone
Cheaper
Lose an AZ → lose data
```

When do we use it?

AWS exam keyword:

> **Easily reproducible**

If data can be recreated,

use

**One Zone-IA**

to save money.

# Why the others are wrong

### ❌ Entire bucket → Intelligent-Tiering

Used when

> You **don't know** access patterns.

Here we already know:

* 1 month frequent
* Then infrequent

No need for Intelligent-Tiering.

---

### ❌ Entire bucket → Standard-IA

Works.

But question asks

> **MOST cost-effective**

One Zone-IA is cheaper.

---

### ❌ Prefix → Standard-IA

Works.

But One Zone-IA is cheaper.

Since files are reproducible,

AWS prefers One Zone-IA.

# Storage Class Cheat Sheet

| Requirement                                           | Storage Class        |
| ----------------------------------------------------- | -------------------- |
| Frequently accessed                                   | Standard             |
| Rarely accessed, instant retrieval                    | Standard-IA          |
| Rarely accessed, instant retrieval, reproducible data | **One Zone-IA**      |
| Unknown access pattern                                | Intelligent-Tiering  |
| Archive, retrieval can wait                           | Glacier              |
| Very long-term archive (cheapest)                     | Glacier Deep Archive |

## 10-second exam memory

```text
Frequently used?
→ Standard

Rarely used?
→ Standard-IA

Rarely + Easily Reproducible?
→ One Zone-IA

Unknown usage?
→ Intelligent-Tiering

Archive?
→ Glacier

Very old archive?
→ Deep Archive
```

This storage-class decision tree appears repeatedly in the SAA exam, so memorizing these keywords will help you answer many S3 questions quickly.

7. Question

Category: CSAA – Design High-Performing Architectures

A company collects atmospheric data such as temperature, air pressure, and humidity from different countries. Each site location is equipped with various weather instruments and a high-speed Internet connection. The average collected data in each location is around 500 GB and will be analyzed by a weather forecasting application hosted in Northern Virginia. The Solutions Architect must determine the fastest way to aggregate all the data. An AWS Direct Connect Partner would be too slow for a one-time aggregation.

Which of the following options can satisfy the given requirement?

- Set up an AWS Site-to-Site VPN connection.
- Upload the data to the closest Amazon S3 bucket. Set up a cross-region replication and copy the objects to the destination bucket.
- `Enable Transfer Acceleration in the destination bucket and upload the collected data using Multipart Upload.`
- Use AWS DataSync to transfer large amounts of data.

This is a classic **exam keyword** question. Don't overthink it.

### Step 1: What is the problem?

* Weather stations all over the world 🌍
* Each location has **500 GB** of data
* Need to send all data to **Virginia**
* **One-time transfer**
* They specifically say **Direct Connect is too slow**

The keyword is:

> **Fastest way**

Not cheapest. Not easiest.

---

## Option 1: Site-to-Site VPN ❌

VPN = secure network tunnel.

It is **not designed to speed up** huge uploads.

Think:

> VPN = Security
> NOT Performance

---

## Option 2: Upload to nearest S3 + Cross-Region Replication ❌

This sounds smart.

```
Country
   ↓
Nearest S3
   ↓
Replication
   ↓
Virginia
```

Problem?

You're making the data travel **twice**.

Country → Local S3

then

Local S3 → Virginia

Extra time.

Also replication isn't instant.

---

## Option 3: S3 Transfer Acceleration + Multipart Upload ✅

This is exactly what AWS built it for.

### S3 Transfer Acceleration

Instead of sending data over the normal Internet:

```
Pakistan
      \
       \
Internet ---------> Virginia
```

AWS does this:

```
Pakistan
      ↓
Nearest CloudFront Edge Location
      ↓
AWS Global Backbone (very fast)
      ↓
Virginia S3 Bucket
```

AWS's private network is much faster than the public Internet.

---

### Multipart Upload

Instead of sending one giant 500 GB file:

```
500 GB
```

Split it into pieces:

```
100 GB
100 GB
100 GB
100 GB
100 GB
```

Upload all pieces **in parallel**.

This dramatically speeds up uploads.

---

## Option 4: AWS DataSync ❌

Many students choose this.

DataSync is great for:

* ongoing migrations
* scheduled transfers
* syncing storage repeatedly

But the question says:

> **one-time aggregation**

DataSync requires:

* creating a DataSync task
* configuring source/destination
* (often) deploying a DataSync agent for on-premises systems

That setup adds unnecessary overhead.

The exam expects the AWS-native optimization for **large global uploads to S3**, which is:

**S3 Transfer Acceleration + Multipart Upload**

---

# Easy Memory Trick

### S3 Transfer Acceleration

> **Far away users uploading to S3 quickly**

---

### Multipart Upload

> **Huge files → split into pieces → upload simultaneously**

---

### DataSync

> **Moving/synchronizing storage between locations (ongoing migrations)**

---

### Site-to-Site VPN

> **Secure connectivity**

---

## Exam Cheat Sheet

| Requirement                    | Answer                     |
| ------------------------------ | -------------------------- |
| Upload huge file quickly to S3 | ✅ Multipart Upload         |
| Users are globally distributed | ✅ S3 Transfer Acceleration |
| Continuous storage migration   | ✅ AWS DataSync             |
| Secure on-prem ↔ AWS network   | ✅ Site-to-Site VPN         |

### One-line memory rule

> **Global + Large Upload + Fastest → S3 Transfer Acceleration + Multipart Upload.**

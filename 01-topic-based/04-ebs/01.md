
1. Question

Category: CSAA – Design Secure Architectures

A company is using an On-Demand EC2 instance to host a legacy web application that uses an Amazon Instance Store-Backed AMI. The web application should be decommissioned as soon as possible and hence, you need to terminate the EC2 instance.

When the instance is terminated, what happens to the data on the root volume?

- Data is automatically saved as an EBS volume.
- `Data is automatically deleted.`
- Data is automatically saved as an EBS snapshot.
- Data is unavailable until the instance is restarted.

Excellent start. This is one of the **highest-frequency EBS questions** on the SAA exam.

---

# Step 1: The keyword

The question says:

> **Amazon Instance Store-Backed AMI**

Immediately think:

> **Temporary storage**

---

# What is Instance Store?

Instance Store is **physical storage attached to the EC2 host**.

```text
EC2 Server

+------------------------+
| CPU                    |
| RAM                    |
| Instance Store Disk    |
+------------------------+
```

It is **not** network storage like EBS.

---

# What happens when the EC2 stops?

Instance Store is tied to the **lifetime of the instance**.

```text
EC2 Running
     ↓
Data exists
```

Terminate EC2

```text
EC2 Deleted
     ↓
Instance Store Deleted
     ↓
Data Gone Forever
```

So the answer is

✅ **Data is automatically deleted.**

---

# Compare with EBS

### Instance Store

```text
Launch EC2
      ↓
Write data
      ↓
Terminate EC2
      ↓
Data LOST
```

---

### EBS

```text
Launch EC2
      ↓
Write data
      ↓
Terminate EC2
      ↓
Depends on DeleteOnTermination
```

Normally the **root EBS volume** is deleted when you terminate the instance, but this behavior can be changed.

Instance Store has **no such option**.

---

# Why the other options are wrong

### ❌ Data is automatically saved as an EBS volume

AWS never converts Instance Store into EBS automatically.

---

### ❌ Data is automatically saved as an EBS snapshot

Snapshots are only created if **you explicitly create them**, and **Instance Store cannot be snapshotted** like EBS.

---

### ❌ Data is unavailable until the instance is restarted

Wrong.

The instance is **terminated**, not stopped.

There is no restart.

---

# Exam Trick

Whenever you see

> **Instance Store**

Immediately think

```text
Fast
Temporary
Ephemeral
Deleted when instance terminates
```

---

# Memory Table

| Storage            | Survives Stop?                                                                                  | Survives Termination?                           |
| ------------------ | ----------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **EBS**            | ✅ Yes                                                                                           | ❌ Usually deleted (unless configured otherwise) |
| **Instance Store** | ❌ No (instance-store-backed instances can't simply be stopped and resumed like EBS-backed ones) | ❌ No, data is lost                              |

---

## Exam Memory Trick

> **EBS = Elastic (persistent).**
> **Instance Store = Instant temporary storage (ephemeral).**

If you see **Instance Store** in an exam question, your first thought should be:

> **"If the instance goes away, the data goes away."**

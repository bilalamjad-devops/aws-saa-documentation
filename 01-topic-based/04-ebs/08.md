Category: CSAA – Design Secure Architectures

An investment bank is working with an IT team to handle the launch of the new digital wallet system. The applications will run on multiple EBS-backed EC2 instances which will store the logs, transactions, and billing statements of the user in an S3 bucket. Due to tight security and compliance requirements, the IT team is exploring options on how to safely store sensitive data on the EBS volumes and S3.

Which of the below options should be carried out when storing sensitive data on AWS? (Select TWO.)

- Use AWS Shield and WAF
- `Enable EBS Encryption`
- Create an EBS Snapshot
- Migrate the EC2 instances from the public to private subnet.
- `Enable Amazon S3 Server-Side or use Client-Side Encryption`

This is another **keyword matching** question. The scenario contains lots of extra information (digital wallet, EC2, logs, S3, compliance), but the real requirement is just one sentence.

---

# Read the requirement carefully

> **Safely store sensitive data on EBS volumes and S3**

The keyword is:

> **store sensitive data**

This means:

✅ **Encryption at rest**

Not networking.

Not backups.

Not DDoS protection.

---

# Resource 1: EBS

How do you protect data stored in EBS?

Answer:

✅ **Enable EBS Encryption**

```text
EC2
 │
 ▼
Encrypted EBS Volume
      │
      ▼
AWS KMS
```

---

# Resource 2: S3

How do you protect objects stored in S3?

Answer:

✅ S3 Encryption

AWS gives two choices:

### Server-Side Encryption (SSE)

AWS encrypts the object.

```text
Your File

↓

S3

↓

AWS Encrypts
```

---

### Client-Side Encryption

You encrypt it before uploading.

```text
Your File

↓

You Encrypt

↓

Upload to S3
```

---

# Therefore the two answers are

✅ Enable EBS Encryption

✅ Enable Amazon S3 Server-Side or Client-Side Encryption

---

# Why are the others wrong?

## AWS Shield + WAF

These protect applications.

Example:

```text
Internet

↓

AWS Shield

↓

WAF

↓

ALB

↓

EC2
```

They stop:

* DDoS
* SQL Injection
* XSS

They **do not encrypt stored data**.

---

## EBS Snapshot

Snapshot means:

```text
EBS

↓

Backup
```

Backup ≠ Encryption

A snapshot is for recovery, not for protecting data at rest.

---

## Move EC2 to Private Subnet

A private subnet improves network security because the EC2 isn't directly reachable from the internet.

However, the question asks:

> **How do you safely store sensitive data on EBS and S3?**

Whether the EC2 is in a public or private subnet **does not encrypt the data**.

So it doesn't satisfy the requirement.

---

# Exam Tip

Whenever you see:

* Sensitive data
* Confidential data
* Compliance
* HIPAA
* PCI DSS
* Encryption at rest

Immediately think:

| Service  | First Answer                      |
| -------- | --------------------------------- |
| EBS      | **EBS Encryption (KMS)**          |
| S3       | **SSE or Client-Side Encryption** |
| RDS      | **RDS Encryption (KMS)**          |
| EFS      | **EFS Encryption (KMS)**          |
| DynamoDB | **DynamoDB Encryption (KMS)**     |

---

# Quick Memory Table

| Service                             | Purpose                                    |
| ----------------------------------- | ------------------------------------------ |
| **EBS Encryption**                  | Encrypt EBS volumes                        |
| **S3 Server-Side Encryption (SSE)** | AWS encrypts S3 objects                    |
| **S3 Client-Side Encryption**       | You encrypt before upload                  |
| **EBS Snapshot**                    | Backup and recovery                        |
| **AWS Shield**                      | DDoS protection                            |
| **AWS WAF**                         | Protect web applications (SQLi, XSS, etc.) |
| **Private Subnet**                  | Network isolation, not storage encryption  |

---

## One exam strategy that will help you

When you read a long scenario, ignore the story at first. Find the **actual requirement**. In this question, everything before the last sentence is mostly context. The requirement is simply:

> **"Safely store sensitive data on the EBS volumes and S3."**

Once you identify that, the answer becomes straightforward:

* **EBS → Enable EBS Encryption**
* **S3 → Enable Server-Side Encryption (or Client-Side Encryption)**

Category: CSAA – Design High-Performing Architectures

A technical lead of the Cloud Infrastructure team was consulted by a software developer regarding the required AWS resources of the web application that he is building. The developer knows that an Instance Store only provides ephemeral storage where the data is automatically deleted when the instance is terminated. To ensure that the data of the web application persists, the app should be launched in an EC2 instance that has a durable, block-level storage volume attached. The developer knows that they need to use an EBS volume, but they are not sure what type they need to use.

In this scenario, which of the following is true about Amazon EBS volume types and their respective usage? (Select TWO.)

- `Provisioned IOPS volumes offer storage with consistent and low-latency performance, and are designed for I/O intensive applications such as large relational or NoSQL databases.`
- `Magnetic volumes provide the lowest cost per gigabyte of all EBS volume types and are ideal for workloads where data is accessed infrequently, and applications where the lowest storage cost is important.`
- Spot volumes provide the lowest cost per gigabyte of all EBS volume types and are ideal for workloads where data is accessed infrequently, and applications where the lowest storage cost is important.
- Single root I/O virtualization (SR-IOV) volumes are suitable for a broad range of workloads, including small to medium-sized databases, development and test environments, and boot volumes.
- General Purpose SSD (gp3) volumes with multi-attach enabled offer consistent and low-latency performance, and are designed for applications requiring multi-az resiliency.

This is mostly a **memory question**. For the SAA exam, you don't need to memorize every EBS volume type in detail, but you should know the common ones and what they're used for.

---

# First, remember the three categories

```text
EBS Volumes

├── General Purpose SSD (gp3/gp2)
├── Provisioned IOPS SSD (io1/io2)
└── HDD / Magnetic (st1/sc1, and older Magnetic)
```

---

# 1. General Purpose SSD (gp3) ⭐

This is the **default choice**.

Use it for:

* Web servers
* Boot volumes
* Small databases
* Development/Test
* General workloads

Think:

> **"Most EC2 instances use gp3."**

---

# 2. Provisioned IOPS (io1 / io2) ⭐

Question says:

> Large relational database

or

> NoSQL database

or

> Highest IOPS

Immediately think:

✅ **Provisioned IOPS**

Example:

```text
Oracle Database

↓

Needs very high IOPS

↓

io2
```

This is why Option 1 is correct.

---

# 3. Magnetic (Previous Generation)

AWS still asks about it in older questions.

Characteristics:

* Cheapest
* Slowest
* Infrequent access

Think:

```text
Old files

↓

Rarely accessed

↓

Magnetic
```

Today, AWS generally recommends:

* st1 (Throughput Optimized HDD)
* sc1 (Cold HDD)

instead of Magnetic, but exams may still mention Magnetic.

---

# Why the others are wrong

## Spot Volume

There is no such thing.

There are:

* Spot Instances ✅

There are NOT:

* Spot Volumes ❌

---

## SR-IOV

SR-IOV has nothing to do with storage.

It is about:

```text
EC2 Networking

↓

Enhanced Networking
```

Not EBS.

---

## Multi-Attach on gp3

This is the trick.

Multi-Attach is supported on:

* io1
* io2

NOT gp3.

Also,

Multi-Attach works only within the **same Availability Zone**.

It does **not** provide Multi-AZ resiliency.

---

# Easy Comparison

| Volume Type  | Use Case                                           |
| ------------ | -------------------------------------------------- |
| **gp3**      | Default, web servers, boot volume, small-medium DB |
| **io1/io2**  | Highest IOPS, mission-critical databases           |
| **st1**      | Throughput-intensive workloads (big data, logs)    |
| **sc1**      | Cold data, lowest-cost HDD                         |
| **Magnetic** | Older generation, cheapest, infrequently accessed  |

---

# Exam Tricks

If the question says...

### "Default EBS volume"

➡️ gp3

---

### "Highest IOPS"

➡️ io1/io2

---

### "Database"

➡️ io1/io2 (for high-performance databases) or gp3 (for general databases, depending on the scenario)

---

### "Cheapest"

➡️ Magnetic (older questions) or sc1 (newer AWS terminology)

---

### "Multi-Attach"

➡️ Only io1/io2

---

### "Enhanced Networking"

➡️ SR-IOV (not storage)

## One small update for current AWS

The explanation mentions **Magnetic** because it's based on older exam content. Today, AWS classifies EBS volumes as:

* **gp3/gp2** (General Purpose SSD)
* **io2/io1** (Provisioned IOPS SSD)
* **st1** (Throughput Optimized HDD)
* **sc1** (Cold HDD)

You may still see **Magnetic** in older practice exams, so know it refers to the legacy, low-cost EBS volume type. For the real SAA exam, focus primarily on **gp3**, **io2/io1**, **st1**, and **sc1**.

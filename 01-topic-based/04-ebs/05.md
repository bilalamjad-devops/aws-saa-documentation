Category: CSAA – Design Secure Architectures

A health organization is using a large Dedicated Amazon EC2 instance with multiple Amazon EBS volumes to host its health records web application. To ensure compliance and consistency, the organization uses AWS Systems Manager State Manager to automatically enforce baseline configurations.

The EBS volumes must be encrypted to protect the confidentiality of the data and to comply with the HIPAA (Health Insurance Portability and Accountability Act) standard.

In EBS encryption, which of the following does AWS use to secure data at rest on the volume? (Select TWO.)

- By using Amazon S3 Server-Side Encryption and S3 Client-Side Encryption.
- By using the SSL certificates provided by the AWS Certificate Manager (ACM).
- `By using Amazon-managed keys in AWS Key Management Service (KMS).`
- `By using your own keys in AWS Key Management Service (KMS).`
- By using a password stored in AWS CloudHSM.

This is a **pure memory question**. There is almost no scenario analysis. Just remember how **EBS encryption** works.

---

# Requirement

The company says:

> Encrypt EBS volumes.

Immediately think:

> **AWS KMS**

Not S3.

Not ACM.

Not CloudHSM.

---

# How EBS Encryption Works

```text
EC2
 │
 │ writes data
 ▼
EBS Volume
 │
 │
 ▼
AWS KMS
(Encryption Key)
 │
 ▼
Encrypted Data Stored
```

AWS encrypts:

* ✅ EBS Volumes
* ✅ Boot Volumes
* ✅ EBS Snapshots

using **AES-256** encryption with keys managed by **AWS KMS**.

---

# AWS gives you two choices

## 1. AWS Managed Key (Amazon-managed)

AWS creates and manages the key.

Example:

```text
KMS

↓

aws/ebs
```

You don't manage anything.

Simply enable encryption.

---

## 2. Customer Managed Key (CMK)

You create your own KMS key.

```text
KMS

↓

My-Hospital-Key
```

Useful when:

* rotation policies
* audit requirements
* compliance
* cross-account permissions

---

# Therefore

The two correct answers are:

✅ Amazon-managed keys in AWS KMS

✅ Your own keys in AWS KMS

---

# Why the others are wrong

## S3 Server Side Encryption

Used only for

```text
S3 Bucket
```

Not EBS.

---

## ACM

AWS Certificate Manager provides

```text
SSL/TLS Certificates
```

Example

```
https://hospital.com
```

It encrypts data **in transit**, not EBS data at rest.

---

## CloudHSM

CloudHSM is a Hardware Security Module.

It stores **cryptographic keys**, not passwords.

Also, EBS encryption is designed to use **AWS KMS**.

---

# Exam Memory Table

| Service      | Uses                                                               |
| ------------ | ------------------------------------------------------------------ |
| **KMS**      | Encrypt EBS, S3, RDS, EFS, Secrets, etc.                           |
| **ACM**      | SSL/TLS certificates (HTTPS)                                       |
| **CloudHSM** | Dedicated hardware security module for managing cryptographic keys |
| **S3 SSE**   | Encrypt S3 objects only                                            |

# One thing to memorize for SAA

Whenever you see:

* EBS encryption
* RDS encryption
* EFS encryption
* DynamoDB encryption

Your first thought should be:

> ✅ **AWS KMS**

Then ask:

* AWS-managed key?
* Customer-managed key?

Those are the two choices AWS commonly tests.

## Quick Revision of EBS (all important points)

| Requirement                 | Service/Feature                                                            |
| --------------------------- | -------------------------------------------------------------------------- |
| Persistent block storage    | **EBS**                                                                    |
| Temporary block storage     | **Instance Store**                                                         |
| Backup EBS                  | **EBS Snapshot**                                                           |
| Automate snapshots          | **Amazon DLM**                                                             |
| Centralized backups         | **AWS Backup**                                                             |
| Encrypt EBS                 | **AWS KMS**                                                                |
| Encryption by default       | **Region-specific setting**                                                |
| EBS Snapshot storage        | Managed by EBS (internally stored by AWS), **not managed like S3 objects** |
| Archive normal backup files | **S3 Lifecycle → Glacier**                                                 |
| EBS and EC2                 | Must be in the **same Availability Zone**                                  |

### A small correction to keep in mind

The explanation says "Amazon-managed keys." In AWS documentation today, you'll typically see the terms:

* **AWS managed KMS key** (for example, `aws/ebs`)
* **Customer managed KMS key (CMK)**

Both are KMS keys, and both can be used for EBS encryption. The exam may use either wording, but the underlying concept is the same: **EBS encryption uses KMS keys.**

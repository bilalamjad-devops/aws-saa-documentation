A company is implementing its Business Continuity Plan. As part of this initiative, the IT Director instructed the IT team to set up an automated backup of all the Amazon EBS volumes attached to the company’s Amazon EC2 instances. The solution must be implemented as soon as possible and should be both cost-effective and simple to maintain.

What is the fastest and most cost-effective solution to automatically back up all of the EBS volumes?

- For an automated solution, create a scheduled job that calls the "create-snapshot" command via the AWS CLI to take a snapshot of production EBS volumes periodically.
- Use an EBS snapshot retention rule in AWS Backup to automatically manage snapshot retention and expiration.
- Set your Amazon Storage Gateway with EBS volumes as the data source and store the backups in your on-premises servers through the storage gateway.
- `Use Amazon Data Lifecycle Manager (Amazon DLM) to automate the creation of EBS snapshots.`

# Option 1

> Scheduled job + AWS CLI

Example:

```
Every night 12 AM

↓

cron job

↓

aws ec2 create-snapshot
```

Will it work?

✅ Yes

But...

You have to maintain:

* cron
* Lambda/EC2
* IAM permissions
* scripts
* monitoring

AWS exam always prefers managed services.

---

# Option 2

> AWS Backup

AWS Backup can absolutely do EBS backups.

It supports:

* EBS
* EFS
* RDS
* DynamoDB
* FSx
* etc.

Think of AWS Backup as:

> **One backup service for many AWS services.**

Example:

```
AWS Backup

├── EBS
├── RDS
├── DynamoDB
├── EFS
├── FSx
```

Very useful when a company wants one centralized backup solution.

---

```
On-premises

↓

AWS
```

```
AWS

AWS
```

Immediately eliminate it.

# Option 4 ✅

> Amazon Data Lifecycle Manager (DLM)

This service exists specifically for:

```
EBS Volume

↓

Automatic Snapshot

↓

Retention

↓

Delete old snapshots
```

No scripting.

No cron.

No maintenance.

Exactly what the question wants.

---

# What is Amazon DLM?

Think of it as:

```
EBS Specialist
```

It only worries about EBS.

Example:

```
Policy

Every day at 12 AM

↓

Create Snapshot

↓

Keep 30 snapshots

↓

Delete older ones
```

Done.

---

# AWS Backup vs DLM

This is the important exam comparison.

| Amazon DLM                   | AWS Backup                  |
| ---------------------------- | --------------------------- |
| Only EBS snapshots           | Many AWS services           |
| Very simple                  | Centralized backup          |
| Automatic snapshot lifecycle | Backup plans & vaults       |
| Best for only EBS            | Best for enterprise backups |

## SAA Cheat Sheet (so far)

| Requirement                                         | Answer                                    |
| --------------------------------------------------- | ----------------------------------------- |
| Persistent block storage                            | EBS                                       |
| Temporary block storage                             | Instance Store                            |
| Automatic EBS snapshots                             | Amazon DLM                                |
| Centralized backups (EBS, RDS, EFS, DynamoDB, etc.) | AWS Backup                                |
| Archive S3 data                                     | Glacier Flexible Retrieval / Deep Archive |
| Point-in-time backup of EBS                         | EBS Snapshot                              |

---

You're now at the point where EBS questions are becoming pattern recognition:

* **EBS volume** → persistent block storage.
* **EBS Snapshot** → backup of an EBS volume.
* **Amazon DLM** → automate EBS snapshot creation, retention, and deletion.
* **AWS Backup** → centralized backup management across multiple AWS services.
* **S3 Lifecycle** → move S3 objects (not EBS snapshots) to Glacier storage classes automatically.

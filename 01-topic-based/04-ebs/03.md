3. Question

Category: CSAA – Design Secure Architectures

A company has several unencrypted Amazon EBS snapshots in its Amazon VPC. The Solutions Architect must ensure that all of the new EBS volumes restored from the unencrypted snapshots are automatically encrypted.

What should be done to accomplish this requirement?

- Enable the EBS Encryption By Default feature for specific EBS volumes.
- Launch new EBS volumes and specify the symmetric encryption AWS KMS keys for encryption.
- Launch new EBS volumes and encrypt them using asymmetric AWS KMS keys.
- `Enable the EBS Encryption By Default feature for the AWS Region.`

This is a classic **SAA exam trap**. The exam is checking whether you know the scope of **EBS Encryption by Default**.

---

# Step 1: Understand the scenario

Current situation:

```text
Unencrypted Snapshot
        |
Create New Volume
        |
❌ Unencrypted
```

Requirement:

> Every **new** EBS volume created from those snapshots must be **automatically encrypted**.

Notice the keyword:

> **Automatically**

That immediately rules out any manual process.

---

# The solution

Enable:

✅ **EBS Encryption by Default**

But where?

👉 **For the AWS Region**

---

# How it works

Before enabling:

```text
Snapshot (unencrypted)

↓

New EBS Volume

↓

Unencrypted
```

After enabling Encryption by Default:

```text
Snapshot (unencrypted)

↓

Create New Volume

↓

AWS automatically encrypts it
```

You don't have to remember to check an encryption box each time.

---

# Very important exam fact

**Encryption by Default is Region-specific.**

Example:

```text
us-east-1

✅ Enabled
```

Every new EBS volume in **us-east-1** is encrypted.

But:

```text
eu-west-1

❌ Not Enabled
```

Volumes there are **not** automatically encrypted.

Each Region has its own setting.

---

# Why the other options are wrong

## ❌ Enable Encryption by Default for specific EBS volumes

You cannot do this.

Encryption by Default works at the **Region level**, not per volume.

Think:

```text
AWS Region

↓

All new EBS volumes
```

Not:

```text
Volume A ✔

Volume B ✘
```

---

## ❌ Launch new volumes and specify KMS key

This works technically.

But it's **manual**.

The requirement says:

> Automatically

So the exam prefers:

> Encryption by Default.

---

## ❌ Use asymmetric KMS keys

EBS does **not** support asymmetric keys.

EBS only supports:

✅ Symmetric KMS keys

---

# Another important concept

Can AWS encrypt an existing unencrypted volume?

❌ No.

Instead you do something like:

```text
Old Unencrypted Volume

↓

Snapshot

↓

Create New Encrypted Volume
```

or enable **Encryption by Default**, and newly created volumes will be encrypted automatically.

---

# Memory Sheet

| Requirement                           | Answer                                            |
| ------------------------------------- | ------------------------------------------------- |
| Automatically encrypt new EBS volumes | ✅ EBS Encryption by Default                       |
| Scope                                 | ✅ Region                                          |
| Existing volumes become encrypted?    | ❌ No                                              |
| Existing snapshots become encrypted?  | ❌ No                                              |
| New volumes from old snapshots        | ✅ Encrypted (if Encryption by Default is enabled) |
| KMS key type                          | ✅ Symmetric only                                  |
| Asymmetric KMS                        | ❌ Not supported                                   |

---

# Exam Memory Trick

When you see:

> **Automatically encrypt every new EBS volume**

Immediately think:

> **EBS Encryption by Default (Region setting).**

When you see:

> **Encrypt an existing unencrypted EBS volume**

Think:

> **Create a new encrypted volume (often via snapshot), since existing volumes can't simply be converted in place.**

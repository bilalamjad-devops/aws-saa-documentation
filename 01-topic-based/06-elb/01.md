1. Question

Category: CSAA – Design Resilient Architectures

A company plans to host a movie streaming app in AWS. The chief information officer (CIO) wants to ensure that the application is highly available and scalable. The application is deployed to an AWS Auto Scaling group of Amazon EC2 instances on multiple AZs with NAT gateways configured for outbound internet traffic. A load balancer must be configured to distribute incoming requests evenly to all EC2 instances across multiple Availability Zones.

Which of the following features should the Solutions Architect use to satisfy these criteria?

- AWS Direct Connect SiteLink
- Path-based Routing
- Amazon VPC IP Address Manager (IPAM)
- `Cross-zone load balancing`

Excellent. ELB is one of the most important SAA topics. This first question tests **Cross-Zone Load Balancing**, which is asked frequently.

---

# Step 1: Understand the Architecture

```
                   Users
                     │
               Load Balancer
              /             \
        AZ-A Node        AZ-B Node
          │                  │
     EC2-1  EC2-2      EC2-3 EC2-4 EC2-5 EC2-6
```

Suppose:

* AZ-A has **2 EC2**
* AZ-B has **4 EC2**

---

# Without Cross-Zone Load Balancing

Each Load Balancer node only sends traffic **inside its own AZ**.

```
Users
   │
──────────────
LB Node A        LB Node B
   │                │
EC2-1 EC2-2     EC2-3 EC2-4 EC2-5 EC2-6
```

Imagine:

100 requests arrive.

Route 53 sends:

* 50 requests → LB Node A
* 50 requests → LB Node B

Now:

LB Node A has only **2 EC2**

```
50 requests

EC2-1 → 25

EC2-2 → 25
```

LB Node B has **4 EC2**

```
50 requests

EC2-3 → 12.5

EC2-4 → 12.5

EC2-5 → 12.5

EC2-6 → 12.5
```

Notice the problem.

AZ-A servers are overloaded.

AZ-B servers are underutilized.

---

# With Cross-Zone Load Balancing

Every LB node can send traffic to **every EC2**.

```
               LB Node A
           /   |   |   |   \
        EC2-1 ...        EC2-6

               LB Node B
           /   |   |   |   \
        EC2-1 ...        EC2-6
```

100 requests

6 EC2

Each receives about:

```
100 / 6 ≈ 16.7 requests
```

Balanced.

Exactly what the company wants.

---

# Exam Keyword

Whenever you read:

* distribute traffic evenly
* multiple AZs
* uneven instances
* load evenly
* equal traffic

Think immediately:

> **Cross-Zone Load Balancing**

---

# Which Load Balancers Support It?

### Application Load Balancer (ALB)

✅ Always enabled

You cannot disable it.

---

### Network Load Balancer (NLB)

❌ Disabled by default

You can enable it.

---

### Gateway Load Balancer (GWLB)

❌ Disabled by default

You can enable it.

---

# Other Options

### Path-Based Routing ❌

Example:

```
/images  → Target Group A

/api     → Target Group B

/admin   → Target Group C
```

This routes based on the URL path, not for balancing traffic evenly across Availability Zones.

---

### AWS Direct Connect SiteLink ❌

Used for connecting on-premises networks over the AWS backbone.

Nothing to do with ELB.

---

### Amazon VPC IPAM ❌

Manages IP address allocation.

Nothing related to traffic distribution.

---

# Quick Memory Trick

The exam wording often says:

> "distribute incoming requests **evenly** across EC2 instances in **multiple Availability Zones**."

This is almost a direct hint for:

✅ **Cross-Zone Load Balancing**

---

# One Small Note for the Exam

The explanation mentions NAT Gateways, but they are just background information. They provide outbound internet access for private instances and have nothing to do with how the load balancer distributes incoming traffic. Don't let that distract you.

---

## 📌 What to remember

* **Cross-Zone Load Balancing** = Every load balancer node can send requests to **all registered targets** in **all enabled AZs**.
* **Without it** = Each load balancer node only sends traffic to targets in **its own AZ**.
* **ALB** → Cross-zone is **always enabled**.
* **NLB & GWLB** → Cross-zone is **disabled by default**, but you can enable it.

This is one of the highest-frequency ELB concepts in the SAA exam, so it's worth remembering well.

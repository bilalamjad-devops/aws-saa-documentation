4. Question

Category: CSAA – Design High-Performing Architectures

A fast food company is using AWS to host its online ordering system which uses an Auto Scaling group of Amazon EC2 instances deployed across multiple Availability Zones with an Application Load Balancer in front. To better handle the incoming traffic from various digital devices, a new routing logic is being considered. Requests with URLs matching /api/android should be directed to a target group named “Android-Target-Group”. Conversely, requests which have a URL of /api/ios are forwarded to another separate target group named “iOS-Target-Group”.

How can this change be implemented in AWS?

- Use host conditions to define rules that forward requests to different target groups based on the hostname in the host header. This enables you to support multiple domains using a single load balancer.
- Replace your ALB with a Gateway Load Balancer then use path conditions to define rules that forward requests to different target groups based on the URL in the request.
- Replace your ALB with a Network Load Balancer then use host conditions to define rules that forward requests to different target groups based on the URL in the request.
- `Use path conditions to define rules that forward requests to different target groups based on the URL in the request.`

This question is testing **Path-Based Routing**, one of the most common ALB topics in the SAA exam.

---

# Step 1: Read the Requirement

The important part is:

> `/api/android`

and

> `/api/ios`

Notice these are **paths in the URL**.

Immediately think:

✅ **Path-Based Routing**

---

# How Path-Based Routing Works

Suppose users access:

```text
https://foodapp.com/api/android
```

ALB checks the path.

Rule:

```text
IF Path = /api/android/*
```

↓

Send to

```text
Android-Target-Group
```

---

Another user visits:

```text
https://foodapp.com/api/ios
```

Rule:

```text
IF Path = /api/ios/*
```

↓

Send to

```text
iOS-Target-Group
```

Diagram:

```text
                    ALB
                     │
        ┌────────────┴────────────┐
        │                         │
Path=/api/android/*      Path=/api/ios/*
        │                         │
Android Target Group      iOS Target Group
```

---

# Why ALB?

Application Load Balancer understands HTTP requests.

It can inspect:

* URL path ✅
* Hostname ✅
* HTTP headers ✅
* Query strings ✅
* HTTP methods ✅

This is called **Layer 7 routing**.

---

# Difference Between Path and Host Routing

Many students confuse these.

## Path-Based Routing

Looks at **everything after the domain**.

Example:

```text
example.com/api

example.com/images

example.com/login
```

The **domain stays the same**.

Only the path changes.

---

## Host-Based Routing

Looks at the **domain name**.

Example:

```text
api.example.com

admin.example.com

shop.example.com
```

Here, the hostname changes.

Diagram:

```text
                ALB
                 │
     ┌───────────┴───────────┐
     │                       │
api.example.com      admin.example.com
     │                       │
API Servers          Admin Servers
```

---

# Why Other Options Are Wrong

### Host Conditions ❌

The question never changes the hostname.

It only changes:

```text
/api/android

/api/ios
```

So host-based routing doesn't fit.

---

### Gateway Load Balancer ❌

GWLB is for:

* Firewalls
* IDS
* IPS
* Security appliances

It does **not** support path-based routing.

---

### Network Load Balancer ❌

NLB works at Layer 4.

It forwards TCP/UDP traffic.

It cannot inspect URLs like:

```text
/api/android
```

or

```text
/api/ios
```

# 🎯 Easy Trick to Remember

Ask yourself one question:

**What is changing?**

If **the domain name changes**, for example:

```text
api.example.com
admin.example.com
```

➡️ **Host-Based Routing**

If **the part after the slash changes**, for example:

```text
example.com/api
example.com/login
example.com/orders
```

➡️ **Path-Based Routing**

This simple check is enough to solve almost every ALB routing question on the SAA exam.


7. Question

Category: CSAA – Design Secure Architectures

A company is hosting its web application in an Auto Scaling group of Amazon EC2 instances behind an Application Load Balancer. Recently, the Solutions Architect identified a series of SQL injection attempts and cross-site scripting attacks to the application, which had adversely affected its production data.

Which of the following should the Architect implement to mitigate this kind of attack?

- `Set up security rules that block SQL injection and cross-site scripting attacks in AWS Web Application Firewall (AWS WAF). Associate the rules to the Application Load Balancer.`
- Using AWS X-Ray to detect and block SQL injection and cross‑site scripting attacks.
- Using AWS Firewall Manager, set up security rules that block SQL injection and cross-site scripting attacks. Associate the rules to the Application Load Balancer.
- Block all the IP addresses where the SQL injection and cross-site scripting attacks originated using the Network Access Control List.

     │

WAF filters malicious HTTP/HTTPS requests **before** they reach your application.

---

# What does AWS WAF protect against?

AWS WAF protects web applications from attacks such as:

* ✅ SQL Injection (SQLi)
* ✅ Cross-Site Scripting (XSS)
* ✅ Bad bots
* ✅ Malicious IPs
* ✅ Rate limiting (DDoS-like request floods)
* ✅ Custom HTTP request filtering

---

# Why is WAF correct?

The attacks happen at the **application layer (Layer 7)**.

Examples:

```
?id=1 OR 1=1
```

```
<script>alert("hack")</script>
```

A firewall like NACL or Security Group only sees:

* Source IP
* Destination IP
* Port
* Protocol

It **cannot inspect HTTP requests** to determine whether they contain SQL injection or XSS payloads.

WAF **can inspect HTTP/HTTPS content**, making it the correct service.

---

# Why not Firewall Manager?

Many students confuse these.

### AWS WAF

Actually blocks attacks.

### AWS Firewall Manager

Manages WAF rules across many accounts.

Think of it like this:

```
Firewall Manager
        │
        ▼
   manages multiple
        ▼
      AWS WAFs
```

Firewall Manager does **not** replace WAF.

Exam trick:

> If the question asks **"block SQL Injection"**

Answer:

✅ WAF

NOT Firewall Manager.

---

# Why not X-Ray?

AWS X-Ray is for:

* Distributed tracing
* Debugging
* Performance analysis
* Finding bottlenecks

It does **not** provide security filtering.

---

# Why not NACL?

NACL works at the subnet level.

It filters:

* IP
* Port
* Protocol

It cannot inspect:

```
GET /login?id=' OR 1=1
```

or

```
<script>alert()</script>
```

So NACL cannot stop SQL injection or XSS.

---

# WAF vs NACL

| AWS WAF               | NACL              |
| --------------------- | ----------------- |
| Layer 7               | Layer 3/4         |
| HTTP/HTTPS inspection | IP/Port filtering |
| SQL Injection         | ❌                 |
| XSS                   | ❌                 |
| Rate limiting         | ❌                 |
| URI/Header inspection | ❌                 |

# ELB Security Decision Table

| Requirement                         | Service                |
| ----------------------------------- | ---------------------- |
| SQL Injection                       | AWS WAF                |
| Cross-Site Scripting (XSS)          | AWS WAF                |
| Too many requests from one IP       | WAF Rate-Based Rule    |
| Block IP/Subnet                     | NACL or Security Group |
| DDoS protection                     | AWS Shield             |
| Manage WAF across many AWS accounts | AWS Firewall Manager   |

6. Question

Category: CSAA – Design Secure Architectures

A company has a web application hosted on a fleet of EC2 instances located in two Availability Zones that are all placed behind an Application Load Balancer. As a Solutions Architect, you have to add a health check configuration to ensure your application is highly-available.

Which health checks will you implement?

- ICMP health check
- FTP health check
- TCP health check
- `HTTP or HTTPS health check`

This is a very common SAA exam question. The key is to remember that **each Load Balancer supports different health check protocols**.

# Rule to Remember

| Load Balancer                       | Health Check              |
| ----------------------------------- | ------------------------- |
| **ALB (Application Load Balancer)** | **HTTP / HTTPS**          |
| **NLB (Network Load Balancer)**     | **TCP, HTTP, HTTPS**      |
| **CLB (Classic Load Balancer)**     | **TCP, HTTP, HTTPS, SSL** |

---

Architecture:

   │       │
EC2      EC2

The question says:

> **Application Load Balancer**

Once you see **ALB**, immediately think:

* HTTP
* HTTPS
* Path routing
* Host routing
* gRPC
* WebSockets

And for health checks:

> ✅ HTTP or HTTPS only

---

## Why not TCP?

TCP health checks only verify:

* Can I establish a TCP connection?

They **cannot verify** whether your web application is actually working.

Example:

Your Apache/Nginx process is accepting TCP connections, but every request returns:

```
500 Internal Server Error
```

A TCP health check would still pass.

An HTTP health check can request:

```
/health
```

and expect:

```
HTTP 200 OK
```

If the application is broken, the target is marked **Unhealthy**.

---

## Why not ICMP?

ICMP is simply a ping.

ALB never uses ICMP health checks.

---

## Why not FTP?

ALB only supports web traffic.

FTP is not supported.

---

# Easy Memory Trick

### ALB = Web

Therefore everything is web-related.

* HTTP
* HTTPS
* HTTP Health Check
* HTTPS Health Check
* Path Routing
* Host Routing
* gRPC
* WebSockets

---

### NLB = Network

Therefore everything is network-related.

* TCP
* UDP
* TLS
* TCP Health Check

---

# Exam Shortcut

Whenever you read:

* **Application Load Balancer**
* **Website**
* **HTTP**
* **HTTPS**

Think:

> **Health Check = HTTP/HTTPS**

Whenever you read:

* **Network Load Balancer**
* **TCP**
* **UDP**
* **Gaming**
* **VoIP**

Think:

> **Health Check = TCP (or HTTP/HTTPS if configured)**

2. Question

Category: CSAA – Design Resilient Architectures

A DevOps Engineer is required to design a cloud architecture in AWS. The Engineer is planning to develop a highly available and fault-tolerant architecture consisting of an Elastic Load Balancer and an Auto Scaling group of EC2 instances deployed across multiple Availability Zones. This will be used by an online accounting application that requires path-based routing, host-based routing, and bi-directional streaming using Remote Procedure Call (gRPC).

Which configuration will satisfy the given requirement?

- Configure a Network Load Balancer in front of the auto-scaling group. Create an AWS Global Accelerator accelerator and set the load balancer as an endpoint.
- Configure a Network Load Balancer in front of the auto-scaling group. Use a UDP listener for routing.
- Configure a Gateway Load Balancer in front of the auto-scaling group. Ensure that the IP Listener Routing uses the GENEVE protocol on port 6081 to allow gRPC response traffic.
- `Configure an Application Load Balancer in front of the auto-scaling group. Select gRPC as the protocol version.`

This is a **very important ELB question** because it teaches you **when to choose ALB vs NLB vs GWLB**.

---

# Step 1: Read the Requirements

The question mentions:

* ✅ Highly available
* ✅ Auto Scaling
* ✅ **Path-based routing**
* ✅ **Host-based routing**
* ✅ **gRPC**

Ignore everything else.

The answer depends on these three keywords.

---

# Keyword 1: Path-Based Routing

Example:

```
example.com/images
```

↓

Image Service

---

```
example.com/api
```

↓

API Service

---

```
example.com/orders
```

↓

Order Service

Only **Application Load Balancer (ALB)** can do this.

---

# Keyword 2: Host-Based Routing

Example:

```
api.company.com
```

↓

API Servers

---

```
admin.company.com
```

↓

Admin Servers

---

```
shop.company.com
```

↓

Shopping Servers

Again,

Only **ALB** supports Host-based Routing.

---

# Keyword 3: gRPC

This is the biggest clue.

gRPC is a modern communication protocol used by microservices.

Example:

```
Frontend
     │
     ▼
Order Service
     │
     ▼
Payment Service
     │
     ▼
Inventory Service
```

Instead of using REST APIs,

they communicate using

**gRPC**

AWS supports gRPC **only on ALB**.

So as soon as you see:

> **gRPC**

Immediately think

✅ **Application Load Balancer**

# Therefore

The answer becomes obvious.

```
ALB

✓ Path Routing

✓ Host Routing

✓ gRPC
```

---

# Why NLB is Wrong

Network Load Balancer works at

**Layer 4**

It only understands

* TCP
* UDP
* TLS

It does **not** inspect URLs or host names.

It cannot say

```
/images

/api
```

It simply forwards packets.

It also does **not** support gRPC in the SAA exam context.

---

# Why Gateway Load Balancer is Wrong

Gateway Load Balancer is for

Security Appliances.

Examples

```
Firewall

IDS

IPS

Deep Packet Inspection
```

Think

```
Traffic

↓

Firewall

↓

EC2
```

Not web applications.

---

# Why Global Accelerator is Wrong

Many students confuse this.

Global Accelerator

does **NOT**

replace a load balancer.

It simply makes users reach AWS faster.

Example

Pakistan User

↓

Nearest AWS Edge

↓

AWS Global Network

↓

Your ALB/NLB

It improves latency.

It does **NOT**

provide

* Path Routing
* Host Routing
* gRPC

## ⭐ Memory Trick

You can remember the three AWS load balancers like this:

* **ALB = Smart** → Understands URLs, hostnames, HTTP, gRPC, and can make routing decisions.
* **NLB = Fast** → Moves network packets (TCP/UDP) with very high performance and low latency.
* **GWLB = Security** → Sends traffic through firewalls and other virtual security appliances.

For the SAA exam, **Path-based routing + Host-based routing + gRPC** is almost a giveaway for **Application Load Balancer**.
